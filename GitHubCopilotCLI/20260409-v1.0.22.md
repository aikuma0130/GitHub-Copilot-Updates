# GitHub Copilot CLI v1.0.22 - 2026-04-09 アップデート情報

## リリース日

2026-04-09

## GitHub Copilot CLI バージョン

1.0.22

## 概要

v1.0.22 では、カスタムエージェントへの skills フィールドの追加やプラグインのインストール後メッセージ表示など、拡張性に関する新機能が追加されました。また、MCP 設定ソースが `.mcp.json` に一本化され、レンダリングパフォーマンスの改善やサブエージェントの安定性向上も行われています。

## 変更点

### 新機能
- カスタムエージェントが `skills` フィールドを宣言して、起動時にスキルコンテンツをエージェントコンテキストに事前ロードできるようになった
- プラグインがインストール後にセットアップ手順を含むメッセージを表示できるようになった
- サブエージェントの深度と同時実行数の制限が追加され、エージェントの暴走的な生成を防止できるようになった
- 既に別のCLIまたはアプリケーションで使用中のセッションを再開しようとした際に警告が表示されるようになった

### 改善
- 非標準の JSON スキーマを持つ MCP ツールが、すべてのモデルプロバイダーとの互換性のためにサニタイズされるようになった
- MCP および拡張ツールからの大きな画像の処理が改善された
- 新しいシンプルなインラインレンダラーによりレンダリングパフォーマンスが向上した
- リモートセッションがポリシーによりブロックされた場合、組織管理者に連絡するよう明確なメッセージが表示されるようになった
- スラッシュコマンドピッカーがテキスト入力の上に表示されるようになり、レイアウトがより安定した
- プラグインがセッション間で有効状態を維持し、ユーザー設定に基づいて起動時に自動インストールされるようになった

### 削除
- `.vscode/mcp.json` および `.devcontainer/devcontainer.json` が MCP サーバー設定ソースから削除され、CLI は `.mcp.json` のみを読み込むようになった（`.vscode/mcp.json` が検出され `.mcp.json` がない場合は移行ヒントが表示される）

### バグ修正
- サブエージェントのアクティビティでツール名が重複して表示される問題が修正された（例: "view view the file..."）
- Anthropic モデルを BYOM/BYOK 設定で使用する際に、パーミッションチェックやその他のフックが正しく動作するようになった
- V8 エンジンの書記素分割バグの影響を受けるシステムで CLI がクラッシュする問題が修正された
- sessionStart および sessionEnd フックがインタラクティブモードでプロンプトごとではなくセッションごとに1回だけ発火するようになった
- プラグインエージェントがフロントマターで指定されたモデルを正しく使用するようになった
