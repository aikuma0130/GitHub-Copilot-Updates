# GitHub Copilot CLI v1.0.25 - 2026-04-13 アップデート情報

## リリース日

2026-04-13

## GitHub Copilot CLI バージョン

1.0.25

## 概要

v1.0.25 では、MCP サーバーのレジストリからのインストール機能やリモートセッション制御機能など複数の新機能が追加されました。また、各種コマンドの使い勝手の改善や UI の不具合修正も含まれています。

## 変更点

### 新機能
- MCP サーバーをレジストリからガイド付き設定で直接 CLI にインストールできるようになった
- ACP クライアントがセッションの開始・読み込み時に MCP サーバー（stdio、HTTP、SSE）を提供できるようになった
- `/env` コマンドが追加され、読み込まれた環境の詳細（インストラクション、MCP サーバー、スキル、エージェント、プラグイン）を表示できるようになった
- `--remote` フラグまたは `/remote` コマンドで CLI セッションをリモート制御できるようになった
- Alt+D でカーソル前方の単語を削除できるようになった

### 改善
- 解決済みモデルがセッション履歴に保持され、アクティブなターン中のモデル変更が延期されるようになった
- `--config-dir` フラグがアクティブモデルの選択時に正しく尊重されるようになった
- `/share` がカスタム出力パスに拡張子がない場合、正しいファイル拡張子（.md または .html）を自動付与するようになった
- `/add-dir` が相対パス（例: ./src、../sibling）を受け付け、絶対パスに解決するようになった
- `/logout` が gh CLI、PAT、API キー、環境変数でサインインしている場合に警告を表示するようになった（/logout は OAuth セッションのみを管理するため）
- `/share html` が file:// URL を表示し、Ctrl+X O でファイルを直接開けるようになった
- MCP クライアントがサーバーハンドシェイク時に正しい CLI バージョンを報告するようになった
- MCP リモートサーバー接続が一時的なネットワーク障害時に自動リトライするようになった

### バグ修正
- `/resume` のセッション検索が失敗した後に Esc キーが正しく動作するようになった
- カスタムインストラクションファイルが & などの特殊文字を正しく保持するようになった
- スキルピッカーのリストがターミナルの高さを超えた場合に正しくスクロールするようになった
- スキルインストラクションが会話のターンをまたいで正しく保持されるようになった
- Share Research の目次サイドバーのアンカーリンクがページ内で正しくナビゲートするようになった
