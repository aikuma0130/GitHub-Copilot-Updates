# GitHub Copilot CLI v1.0.26 - 2026-04-14 アップデート情報

## リリース日

2026-04-14

## GitHub Copilot CLI バージョン

1.0.26

## 概要

v1.0.26 では、プラグインフックへの環境変数の追加やキーボードショートカットの拡充などの新機能が含まれています。また、セキュリティ改善や重複インストラクションの排除によるトークン効率化、多数のバグ修正が行われています。

## 変更点

### 新機能
- プラグインフックが PLUGIN_ROOT、COPILOT_PLUGIN_ROOT、CLAUDE_PLUGIN_ROOT 環境変数でプラグインのインストールディレクトリを受け取れるようになった
- ctrl+o ですべてのタイムラインエントリを展開できるようになった（ctrl+e と同じ動作）

### 改善
- `--remote` フラグおよび `/remote` コマンドのヘルプテキストで「steering」が「remote control」にリネームされた
- 重複するカスタムインストラクションファイル（例: copilot-instructions.md と CLAUDE.md が同一内容の場合）の送信を回避し、ターンあたりの無駄なトークンを削減
- パーミッションプロンプトの通知フックが、実際にプロンプトがユーザーに表示された場合のみ発火するようになった
- Remote タブが Copilot coding agent タスクを正しく表示し、プルリクエストなしでステアリングをサポートするようになった
- 同期プロンプトのセッションスコープセレクターがより目立つようになり、左右の矢印キーでキーボードナビゲーションが可能になった
- 特定の applyTo パターンを持つインストラクションファイルがフルコンテンツのインライン展開ではなくテーブルに集約され、コンテキストウィンドウの使用量が削減された
- ACP サーバーが localhost のみにバインドするようになり、意図しないネットワーク露出を防止

### バグ修正
- Esc キーが ask_user およびエリシテーションプロンプトを確実に閉じるようになり、スタックしなくなった
- find -exec ブロック内の引数に対する不要なディレクトリアクセスプロンプトが表示されなくなった
- コンテキスト圧縮がチェックポイント境界をまたいでツールコールを分割した場合に、エージェントセッションが回復不能なエラーで失敗しなくなった
- 単一セグメントのスラッシュプレフィックス付きトークン（例: /help、/start）が bash コマンド内でファイルパスとして扱われなくなった
- Anthropic BYOM がイメージファイルを表示する際に画像データを正しく含めるようになった
- マーケットプレイスから 'git' という名前のプラグインをインストールする際に、不正な URL パースが原因で失敗しなくなった
- Enterprise ログインが URL スキームなしのホスト名（例: 'github.example.com'）を受け入れるようになった
- Windows で LSP 言語サーバーが適切なファイル URI パスを使用して正しく初期化されるようになった
- ファイル編集操作の相対パスがセッションの作業ディレクトリに対して解決されるようになった
