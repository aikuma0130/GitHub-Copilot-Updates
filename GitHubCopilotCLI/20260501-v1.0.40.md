# GitHub Copilot CLI v1.0.40 - 2026-05-01 アップデート情報

## リリース日

2026-05-01

## GitHub Copilot CLI バージョン

1.0.40

## 概要

MCP サーバー向けの `client_credentials` OAuth 認証サポートや Azure DevOps リポジトリの自動検出など、多数の新機能が追加されました。また、CLI の起動速度改善やオートパイロットモードの制御強化など、全体的な安定性とユーザビリティが向上しています。

## 変更点

### 新機能
- MCP サーバー向けに `client_credentials` OAuth グラントタイプのサポートを追加し、ブラウザなしでの完全ヘッドレス認証が可能に
- Azure DevOps リポジトリを検出し、GitHub MCP サーバーを自動で無効化
- セッション履歴、ファイル追跡、および `/chronicle` コマンドが全ユーザーに利用可能に
- スキルが ACP クライアントでスラッシュコマンドとして利用可能に（CLI と同等の体験）
- ACP クライアントがエージェント設定オプションからカスタムエージェントの一覧表示と切り替えが可能に

### 改善
- PR ブランチ装飾がモデル名の長さに関係なくフッターで正しく表示されるように
- `/clear` と `/new` でアクティブなカスタムエージェント選択がリセットされるように
- アシスタントの応答がよりスムーズなテキスト出力でストリーミングされるように
- `copilot plugin list` が `copilot plugin update` 実行後に正しいバージョンを表示するように
- プロンプトモードで Ctrl+C を押した際に即座に "Exiting…" を stderr に出力し、シャットダウンの進行状況を可視化
- `/research` がオーケストレーター/サブエージェントモデルを使用し、より徹底的で信頼性の高いディープリサーチ結果を提供
- オートパイロットモードの継続メッセージがデフォルトで5回に制限（`--max-autopilot-continues` で設定可能）
- 自動更新時に古い CLI パッケージバージョンをディスクから自動クリーンアップ
- リモートセッションのステータスラインにローカルコンテキストではなくリモート作業ディレクトリとブランチを表示
- カスタム CA 証明書の非同期読み込みにより CLI の起動速度を改善
- リモートコントロールリンクがタイムラインで「ブラウザで開く」ではなく完全な URL を表示
- ACP クライアント（例: Zed）がマルチステップタスク実行中にエージェントのライブプランを表示
- ステータスラインピッカーにカスタム `statusLine.command` の表示切り替えを追加
- Ctrl+C とダブル Esc でキューに入ったメッセージを一括ではなく1つずつ削除
- スラッシュコマンドの候補表示で前方一致をファジーマッチより上位にランク
- プロンプトモード（`-p`）でリポジトリフックとワークスペース MCP をオプトイン環境変数（`GITHUB_COPILOT_PROMPT_MODE_REPO_HOOKS` / `GITHUB_COPILOT_PROMPT_MODE_WORKSPACE_MCP`）でゲートし、デフォルトで安全な動作を実現

### バグ修正
- サブエージェントが親セッションの設定を継承するのではなく、自身のモデルのツール検索サポートを正しく評価するように修正
- `/new` または `/resume` でセッションを切り替えた際に保留中のメッセージが新しいセッションに引き継がれなくなった
- 大きなファイル添付を送信した際に CLI が CPU 100% でハングする問題を修正
- セッション再開ピッカーで同じ Mission Control バックエンドセッションの重複エントリが表示されなくなった
- セッション再開セレクターの要約が1行で表示され、列幅に合わせて切り詰められるように
- `/update` が再起動後に元の `-i` プロンプトを再送信しなくなった
- 前の CLI プロセスが予期せず終了した後にセッションを再開しても使用中と誤報告されなくなった
- `--config-dir` がプラグインサブコマンドに正しく伝播されるように修正（`--config-dir` は `COPILOT_HOME` に移行予定で非推奨に）
- `/ask` レスポンスダイアログが開いている間もマウス選択が機能し、コンテンツのハイライトとコピーが可能に
- 複数の MCP サーバーが同じ URL を共有しつつ異なる静的 OAuth クライアント ID を使用する場合に OAuth トークンが正しくキャッシュされるように
- MCP ツール名にドットやその他の無効な文字が含まれる場合に正しくサニタイズされるように
