# GitHub Copilot CLI v1.0.41 - 2026-05-05 アップデート情報

## リリース日

2026-05-05

## GitHub Copilot CLI バージョン

1.0.41

## 概要

CLI の起動速度向上、シェル補完の自動インストール、実験的な MCP Tasks サポートなど、パフォーマンスと拡張性に関する多数の改善が含まれるリリースです。また、Windows での安定性向上やファイル編集の信頼性改善など、多くのバグ修正が行われています。

## 変更点

### 新機能
- 実験的な MCP Tasks サポートを追加: `taskSupport: "required"` を持つ MCP ツールがノンブロッキングのバックグラウンドエージェントとして実行され、`list_agents` と `read_agent` で追跡可能に（実験モード有効時に利用可能）
- 非インタラクティブモード（`-p`/`--prompt`）でファイル（画像やネイティブドキュメント）を初期プロンプトに添付する `--attachment` フラグを追加

### 改善
- UI を即座にレンダリングし、認証をバックグラウンドで解決することで CLI の起動速度を向上
- シェル補完（bash、zsh、fish）が初回実行時に自動インストールされ、`copilot update` 後に自動更新されるように
- 引数を受け取るスラッシュコマンドのタブ補完で末尾にスペースが自動追加されるように
- スラッシュコマンドピッカーがコマンドの説明を検索し、一致した文字にアンダーラインを表示
- メモリツールの確認プロンプトでメモリの保存先スコープ（リポジトリまたはユーザー）を表示
- SQL todo タイムラインエントリが INSERT OR IGNORE/REPLACE およびブロック状態の更新に対してより正確に表示
- ストリーミングテキストとシマーアニメーションが低速またはビジーなホストでもスムーズに動作
- ファジーまたはずれた編集ブロックからの回復を改善し、ファイル編集の信頼性を向上

### バグ修正
- Windows でアンチウイルスやファイルシステムロックによる一時的な EPERM エラーが発生した際にパッケージ展開がクラッシュしなくなった
- リモートセッション接続エラーでログイン中のアカウントと対処手順を表示するように
- ask user プロンプトの質問で Markdown フォーマットがレンダリングされるように
- プロンプトモード（`-p`）で拡張機能が読み込まれるように修正（ユーザー拡張機能はデフォルトで読み込み、プロジェクト拡張機能と管理ツールは `GITHUB_COPILOT_PROMPT_MODE_EXTENSIONS=true` が必要）
- アシスタントの応答に不要なシステム通知 XML タグが含まれなくなった
- 大きな出力のガイダンスで設定された grep ツール名が正しく参照されるように
- git SSH URL（例: git@github.com:owner/repo）を使用したプラグインマーケットプレイスの追加が正しく動作するように
- `@` メンション補完が `./` パスで動作し、ディレクトリに末尾スペースを追加しなくなり、ワークスペースルートよりプロジェクトファイルを優先表示
- Node 24.x での V8 クラッシュを回避し Windows での安定性を向上
- Unicode 行区切り文字を含むセッションファイルが正しく読み込まれるように
- 推論エフォートピッカーのヒントテキスト「Esc to cancel」が正しいスペーシングで表示されるように
