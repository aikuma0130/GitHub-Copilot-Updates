# GitHub Copilot CLI v1.0.49 - 2026-05-18 アップデート情報

## リリース日

2026-05-18

## GitHub Copilot CLI バージョン

1.0.49

## 概要

新しいスラッシュコマンド（/chronicle search、/rubber-duck、/memory、/session id、/exit print）の追加、Alpine Linux（musl libc）サポート、MCP 関連の改善、および多数のバグ修正を含む大規模なリリースです。

## 変更点

### 新機能

- /chronicle search サブコマンドを追加し、キーワードやトピックでセッション内容全体を検索可能に
- Alpine Linux（musl libc）での CLI 実行をサポート
- /exit print オプションを追加し、終了前にセッションをターミナルに出力可能に
- /rubber-duck コマンドを追加し、エージェントの現在の作業に対する独立した批評を取得可能に（実験的）
- /session id サブコマンドを追加し、現在のセッション ID の表示とクリップボードへのコピーが可能に
- MCP サーバー向けに `auth.redirectPort` 設定オプションを追加し、OAuth コールバックを固定ポートにピン止め可能に
- /memory on|off|show スラッシュコマンドを追加し、メモリステータスの有効化・無効化・表示が可能に（永続的）
- `copilot plugin update --all` を追加し、インストール済みの全プラグインを一括更新可能に
- 推論エフォートピッカーに「None」オプションを追加し、モデルの推論を無効化可能に
- COPILOT_PLUGIN_DIR_ONLY 環境変数を追加し、自動プラグイン検出を無効化して --plugin-dir 使用時に決定論的なプラグインセットを実現
- 実験的: /mcp search コマンドでレジストリから MCP サーバーを検索・インストール可能に
- 実験的: MCP および外部ツールの遅延読み込みによるツール検索機能

### 改善

- postToolUse フックの additionalContext がサイレントに破棄される代わりにモデルへのシステムメッセージとして注入されるように改善
- /user switch がフェッチ済みのユーザーリストを再利用し、初回オープン時にローディングスピナーを表示するように改善
- 入力プロンプトが空の時に1行に折りたたまれ、入力に応じて自然に拡大するように改善
- 実験的スラッシュコマンドがヘルプダイアログとコマンドピッカーで「(experimental)」と注釈表示されるように改善
- 自動更新がユニバーサルパッケージの代わりに小さいプラットフォーム固有パッケージをダウンロードするように改善
- アシスタント応答内で GitHub issue と PR の参照（owner/repo#number）を自動リンクするように改善
- プロンプトモード（-p）で現在のフォルダが既に信頼されている場合にワークスペース MCP ソースを自動ロードするように改善
- MCP stdio サーバーが一貫性のため 'local' の代わりに 'stdio' と表示するように改善
- ドキュメント添付パスがコンテキストに含まれ、エージェントが貼り付けられたファイルパス（Windows の「パスとしてコピー」入力を含む）を参照可能に
- メモリ権限プロンプトがメモリの閲覧可能者（ユーザースコープまたは特定の `owner/repo`）を表示するように改善。タイムラインエントリにもスコープを表示
- Windows で従来の PowerShell 5.x 使用時に && チェーン命令を避けることで PowerShell 構文エラーを削減

### バグ修正

- ワイド文字（CJK、絵文字）を含む入力時にプロンプト内のマウスクリックがカーソルを正しく配置するように修正
- 静的 OAuth クライアントを使用する MCP サーバーがトークン更新のために登録を正しく永続化するように修正
- すべての編集ツールタイプに対してファイル差分が ACP クライアントに正しく報告されるように修正
- フォルダが既に信頼されている場合にプロンプトモード（-p）で `.github/hooks/` のリポジトリフックがロードされるように修正
- タイムラインエントリの余分な行を修正
- UTF-8 コードページを使用していない Windows ターミナルで罫線文字とブロック文字が正しくレンダリングされるように修正
- `args` フィールドのない MCP サーバー設定が受け入れられ、空の args リストとして扱われるように修正
- tmux セッションでプログレスバーインジケーターが正しく表示されるように修正
- スクロールビューからのテキストコピーが余分な改行やインデントなしでソフトラップされた行を結合するように修正
- 入力フィールドでワイド文字（CJK、絵文字）によるカーソル位置が正しく動作するように修正
- フック（preToolUse、postToolUse、subagentStart、subagentStop）がサブエージェントのツール呼び出しに対して正しく発火するように修正
- --plugin-dir で読み込まれたプラグインがプロンプトモードで利用可能な task(agent_type=...) サブエージェントとして正しく登録されるように修正
- リポジトリコンテキストがない場合にメモリストレージが利用可能なスコープを正しく制限するように修正
- --plugin-dir および --additional-mcp-config が --server / --headless モードで動作するように修正
- コンテンツフィルタリングされたモデル応答が空白のアシスタントターンの代わりに説明を表示するように修正
- 外部ターミナルが ghostty、WezTerm、または kitty の場合に tmux 内で PromptFrame UI がレンダリングされるように修正
- MCP OAuth トークン検索がアクティブセッションに正しくスコープされるように修正
