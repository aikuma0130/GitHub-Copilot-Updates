# GitHub Copilot CLI v1.0.51 - 2026-05-20 アップデート情報

## リリース日

2026-05-20

## GitHub Copilot CLI バージョン

1.0.51

## 概要

セッション管理の強化（`--session-id` によるセッション再開）、セキュリティ関連の新コマンド（/security-review）、MCP フック拡張、およびリモートコマンドのポリシー対応を含むリリースです。

## 変更点

### 新機能

- `--session-id=<id>` オプションを追加し、既知のセッションやタスクの再開、特定の UUID での新規セッション開始が可能に
- /security-review スラッシュコマンドを追加し、コード変更のセキュリティ脆弱性レビューが可能に
- preMcpToolCall フックを追加し、フックプロバイダーが送信 MCP リクエストのメタデータを制御可能に
- /chronicle cost-tips サブコマンドを追加し、パーソナライズされたトークン使用量とコスト削減の推奨を表示可能に
- `terminalProgress` 設定を追加し、OSC 9;4 ターミナルプログレスインジケーターの有効・無効を切り替え可能に
- シークレットスキャンがコミットメッセージと PR 説明文もカバーし、公開前にシークレットを検出・マスクするように拡張

### 改善

- /remote コマンドが組織のリモート制御およびクラウドポリシーからの表示を尊重し、無効時に明確なエラーを表示するように改善
- /remote コマンドがエージェント動作中にも使用可能に
- ターミナルフッターのカスタマイズ可能なステータスラインにモデル、コンテキストウィンドウ、git ブランチなどのセッション情報を表示
- 多数の HTTP ベース MCP サーバーを持つユーザー向けに起動時の MCP ツール読み込みを高速化
- `/memory show` が Copilot Memory の学習・管理に関するドキュメントリンクを表示するように改善
- postToolUse フックが成功したツール結果に additionalContext を注入可能に
- 実験モードインジケーターが1回限りの通知ではなくアプリヘッダーに永続的に表示されるように改善
- ローディングインジケーターの色がアクティブモード（plan、autopilot、shell）に一致するように改善
- 入力トークン使用量にキャッシュを含め、トークンフォーマットを明確化
- ログインプロンプトがトークンストレージが安全でないプレーンテキスト設定ファイルにフォールバックする際に明確に警告するように改善
- GitHub MCP ウェブ検索ツールがツール検索なしで即座に利用可能に
- 入力エリアが3行の上限ではなくターミナルの高さに応じてレスポンシブに拡大するように改善
- リモートセッション起動失敗を --remote で明示的にリクエストされた場合またはユーザー設定で有効な場合のみ表示するように改善

### バグ修正

- 設定ファイルが更新時に無関係な設定キーを蓄積しなくなるように修正
- 別のセッションで認証が行われた場合に OAuth を使用する MCP サーバーが接続を維持するように修正
- リストアイテム内の GFM テーブルとブロッククォートが浮遊する上部ボーダーなしで正しくレンダリングされるように修正
- 使用量ベース課金ユーザーのセッション命名が正しく動作するように修正
- ハイライトされたサブコマンド補完で Enter キーを押すと部分的なコマンドを送信する代わりに選択を挿入するように修正
- チェンジログエントリがない場合にリリース公開時にデフォルトのリリースノートを使用するように修正
- Ctrl+G で起動したエディターがキーストロークを奪ったり二重キー押下を要求しなくなるように修正
- モデルが `description` パラメータを省略した場合でもシェルツール呼び出しが成功するように修正
