# GitHub Copilot CLI v1.0.65 - 2026-06-24 アップデート情報

## リリース日

2026-06-24

## GitHub Copilot CLI バージョン

1.0.65

## 概要

作業ディレクトリの永続化、スキル管理サブコマンドの追加、CI ステータスバーの導入など、セッション管理とワークフロー効率を向上させるアップデートです。

## 変更点

### 新機能

- `/cd` で作業ディレクトリを永続化し、セッション再開時にそのディレクトリに戻り、新しいディレクトリのカスタムエージェントを検出
- CI チェックステータス（passing/running/failing）を現在のブランチに対して表示するオプトインステータスバー項目を追加
- `copilot skill` サブコマンド（`/skill` は `/skills` のエイリアス）を追加し、ファイル・URL・ディレクトリからスキルの一覧表示・追加・削除が可能に
- カスタムステータスラインコマンドを `/settings` に保存可能に
- 上下矢印の履歴検索と Ctrl+R 逆方向検索が通常モードでも過去のシェルコマンドを含むように（`!` を入力せずにシェルコマンドを呼び出し・再実行可能）

### 改善

- スラッシュ接頭辞の文字列引数を持つコマンド（例：`--body "/azp run"`）が不要なファイルシステム権限プロンプトをトリガーしなくなった
- フルスクリーンタイムラインで古いコンテンツがトリミングされた際にアンカーを維持
- CLI 再起動後にオープンキャンバスを自動的に再開
- 非 GitHub テーマ使用時に起動時の GitHub 背景のフラッシュを防止
- Windows でエージェントがフックコマンドを実行したりコマンドパスを解決する際に短時間のコンソールウィンドウがフラッシュしなくなった
- `userPromptSubmitted` フックの `additionalContext` をモデル向けプロンプトに含めるように
- stdio MCP サーバー追加時に Windows パスをそのまま維持
- MCP シャットダウンが進行中のサーバー接続を待機しなくなった
- シャットダウンタイムアウトなしで CLI を再起動
- タイムラインのシェルコマンドからシンタックスハイライトを削除
- BYOK プロバイダー使用時にカスタムエージェントのサブエージェントモデル選択を維持
- `/every` のスケジュールをセッションのメインモデルで解析
- tmux でインライン画像を安定してレンダリング
- `ask_user` のフリーフォームオプションがテキストを折り返し、カーソルの位置を揃えるように
- ストリーミングバイトカウントをキャンセルヒントと別に表示
- セルフペーススケジュールが非アクティブ時のウェイクアップ誤動作をグレースフルメッセージで処理
- サイレント MCP OAuth リフレッシュが付与済みスコープを再利用し、再接続がサインイン状態を維持
