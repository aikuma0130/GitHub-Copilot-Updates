# GitHub Copilot CLI v1.0.66 - 2026-06-30 アップデート情報

## リリース日

2026-06-30

## GitHub Copilot CLI バージョン

1.0.66

## 概要

タイムラインのコンパクト表示の全ユーザー展開、MCP サーバー管理の強化、セッション制限の改善、および多数のバグ修正を含む大規模アップデートです。

## 変更点

### 新機能

- タイムラインをコンパクトな「ハイライトリール」として表示し、ツールと推論の行を1行にまとめる機能を全ユーザーに展開
- Claude Opus 4.8 Fast のサポートを追加し、Claude Opus 4.6 Fast を非推奨に
- MCP 追加/編集フォームで HTTP スタイルの `Key: value` ヘッダーを受け付けるように
- リレーセッションで @ ファイルおよび # GitHub ref の補完を追加
- 現在のプルリクエストリンクをステータスライン項目として表示
- WebSocket Responses リクエストのクォータスナップショットを表示
- macOS の CLI からデスクトップ通知を表示
- `/chronicle skills review` を追加し、提案されたドラフトスキル変更のレビュー（承認・拒否・延期）が可能に
- アテンションプロンプトおよびアイドルセッションのデスクトップ通知を追加
- MCP リストビューから MCP サーバーの有効/無効を切り替えるトグルを追加
- CLI 設定に実験的なレスポンス制限コントロールを追加
- 永続化された dynamicRetrieval 設定（`--dynamic-retrieval skills=<on|off>` フラグ）を追加し、スキルの埋め込みベース検索を有効/無効化
- カスタムエージェントの定義で推論努力レベルを設定可能に
- `/worktree` にタスクを渡してブランチ名を指定し、新しいワークツリーで最初のプロンプトとして実行可能に
- Pull requests タブで各プルリクエストのマージステータスを表示し、`r` キーでキャッシュされたステータスをオンデマンドで更新
- リモートホスト（クラウドおよびリレー）セッションで `/rename` を有効化
- マネージド設定で OpenTelemetry エクスポートを設定可能に
- `/pr auto` がセルフペースループを開始し、CI に合わせて1回ずつ修正を行い PR をグリーンに導く。`/pr automerge` は PR がマージされるまで継続。`/loop` または `/every` から管理・停止可能
- MCP ホストトークンインジェクション OAuth フローのランタイムテレメトリを追加
- `--allow-all-mcp-server-instructions` を追加し、すべての MCP サーバーの指示をシステムプロンプトに含めるオプションを提供
- `/worktree` が引数なしの場合、固定の小さなモデルではなくアクティブモデルを使用して未コミットの変更と最近の会話からブランチ名を生成
- `/worktree` が有効なブランチ名（例：`feature/JIRA-123`）をそのまま維持し、スラグ化（`feature-jira-123`）しないように

### 改善

- インタラクティブセッション中に点滅しないブロックカーソルを使用し、終了時にターミナルのデフォルトカーソルを復元
- LSP サーバーが起動時に2回起動するのを防止
- Windows スタイルのパスフラグメントを含むコマンドのブロッキングを回避
- Copilot がデタッチされたバックグラウンドシェルコマンドの出力読み取りと停止が可能に
- 大きな出力の処理がカスタム出力ディレクトリと無効化設定を尊重
- PR 説明生成が空のアシスタント応答でクラッシュしなくなった
- ファイルシステムの birthtime がない場合に正しいセッション年齢を表示
- GPT モデルの重複する最終アシスタントメッセージを防止
- ターミナルタイトルの更新がより多くのターミナルで動作
- コンパクト Search タイムラインエントリに (sandboxed) バッジを表示
- サンドボックス化されたリンクワークツリーで Git コマンドが動作
- 正確な Anthropic 推論トークン数を表示
- grep と glob がサンドボックス承認後にブロックされた検索を再試行
- ターミナルタイトルをセッションタイトルと GitHub Copilot サフィックスでフォーマット
- tmux でマウスポインタのフリッカーを避けるため同期出力をスキップ
- セッション制限が現在の会話全体に適用され、`/clear` でリセットされ、`sessionLimits` オプションキーを使用
- 除外された組み込みエージェントをエージェント選択から非表示
- 異なるプラグインからの同名スキルの共存を許可
- インテグレーションが CLI ユーザー設定を読み書き可能に
- `/lsp logs` と `read_agent` で LSP サーバーログを表示
- GitHub リポジトリで gh CLI がない場合にインストールを促すプロンプトを表示
- GitHub 添付ファイルバリアントをプロンプトレンダリングに追加
- エクステンショントグルが選択モードを保持
- 添付シェルコマンドのキャンセル後にプロンプトに戻る
- バックグラウンド git ステータスチェックが並行 git コマンドを妨害しなくなった
- 破損したセッション履歴をロード時に復旧
- `/after` と `/every` のスケジュールプロンプトで改行を保持
- 複数行の `/worktree` タスクを開始時にそのまま維持
- `/cd` パス補完で Enter、Escape、Tab の動作を同期
- セッションストア検索とコンテキスト検索の応答性を維持
- コンパクトタイムラインエントリをクリックして展開すると固定され、コンテンツが下方向に表示
- サブエージェントの同時実行数と深度制限を `/settings` で設定可能に（従量課金ユーザー）
- `/share` で Mission Control リンクを使用してセッション共有
- スナップショット作成が一時的な HEAD ルックアップ失敗を再試行
- `/chronicle reindex` の応答性を維持し、タイムラインに進捗を表示
- タブ切り替え時に最後に開いた GitHub issue、PR、gist ビューに戻る
- MCP レジストリエントリのインストール時にパッケージ引数プレースホルダーを解決
- キューされたメッセージがバックグラウンド作業の後ろで停滞しなくなった
- カラーパレット設定を `/settings theme` に統合
- CLI 設定とセッション状態をより確実に保存
- フルスクリーンビューでターミナルの全高を使用
- シェルと検索タイムラインエントリにより明確なアイコンを使用
- ターミナルテキストカラーを GitHub テーマキャンバスに一致
- 作業フッターテキストにアクティブエージェントモードを表示

### バグ修正

- BYOK セッションで Anthropic モデルを使用する際に adaptive-thinking の不一致による HTTP 400 エラーが発生しなくなった
- `read_agent since_turn: 0` が turn 0 を含むすべてのターンを正しく返すように
- MCP サーバー起動時に非 JSON stdout 行をフィルタリング
- トークナイザーのウォームアップをバックグラウンドスレッドで並列実行し起動パフォーマンスを向上
- マネージド設定のフェッチを一時的な接続プールエラー後に再試行
- サンドボックス化された MCP サーバーが終了時にリクエスト中断した際の broken-pipe エラーを表示しなくなった
- MCP ホスト委任接続で OAuth トークンの期限切れまたはより広いスコープが必要な場合に適切に復旧
- CLI の git チェックがオプションロックをスキップし、ビジーなリポジトリでステータスとブランチのルックアップが継続
- インラインフック設定がネストされた Claude スタイルのフックグループを正しく処理
- シークレットフィルタリング中の CLI の応答性を維持
- 検索入力が先頭または末尾にスペースを持つクエリに一致
- ターンをキャンセルした後もアイドルエージェントが利用可能に
- サンドボックスバイパスの警告を表示し、バイパスされたコマンドにラベルを付与
- MCP OAuth 認証済みリモートサーバーでセッション中のトークン期限切れ後に自動復旧（既存の OIDC 再試行動作と一致）
- 非 Main ホームタブがフォーカスされている状態で起動プロンプトが開いた場合の入力無応答を修正
- クロスリポジトリ issue/PR 参照を `owner/repo#number` 形式にガイド
- シャットダウンのティアダウンに時間がかかる場合でも `/restart` が動作
- WSL で `cmd.exe` が PATH にない場合にクリップボードにテキストをコピー
- `COPILOT_HOME` と `--config-dir` が `~/.agents/skills` からスキルをロードしなくなった
- エクステンションモード切替時にエクステンションごとの無効選択を保持
- スクロールビューからの折り返しテキストのコピーでスペーシングが正しく維持
- 起動時にエンジンが起動に失敗した場合にボイスモードが自動的にオフ
- 認証完了前に Ctrl+D でクリーンに終了
- フレーム付きユーザーメッセージが右端で末尾の文字をクリッピングしなくなった
- 終了後にインライン画像がシェルに書き込みを続けなくなった
- `--yolo` セッションでオプトイン MCP 同意プロンプトを自動承認（システム権限プロンプトは引き続き表示）
- MCP サーバーヘッダーを認証変更後に自動的に更新
- LSP コマンドとツールがプロジェクト設定とサーバーパスをより確実に解決
- WSL イメージを Windows 環境変数が利用不可の場合にペースト
