# GitHub Copilot CLI v1.0.69 - 2026-07-07 アップデート情報

## リリース日

2026-07-07

## GitHub Copilot CLI バージョン

1.0.69

## 概要

プラグイン管理の強化（/plugins ダッシュボード追加、拡張機能のホットリロード）、サンドボックスポリシーの柔軟化、MCP サーバー管理の改善、およびセッション性能の向上を含む大型アップデートです。

## 変更点

### 新機能

- /plugins ダッシュボードを追加し、インストール済みプラグインを管理可能に
- /mcp list コマンドを追加し、接続中の MCP サーバーとそのステータスを表示可能に（エージェント動作中も実行可能）
- エージェント動作中に /mcp マネージャーを開いてサーバーの有効/無効を切り替え可能に（追加・編集・削除・再認証はターン終了後）
- LLM ジャッジが許容可能と評価したリクエストを自動承認する auto allow-all モードを追加
- `stayInAutopilot` 設定（デフォルト false）を追加し、オートパイロットタスク完了後もオートパイロットモードを維持可能に
- MCP サーバーへの CLI OAuth コールバックフローによるサインインに対応
- gemini-3.5-flash に最小推論努力度を追加
- /sandbox パスエントリにファイルおよびフォルダ補完を追加
- /delegate --base オプションで別の PR ターゲットブランチを選択可能に（デフォルトは現在のブランチ）

### 改善

- ビルトインファイル編集に (sandboxed) ではなく (sandbox policy) バッジを表示し、ベストエフォートでサンドボックスポリシーに従うことを明確化
- インストール済みプラグイン拡張機能をセッション再起動なしでリロード可能に
- 空のプロンプトからクイックヘルプを開く際に余計な ? を残さないように改善
- /model ピッカーリストにスクロールバーを表示
- /lsp test で無効なサーバーを disabled として表示
- ビルトインファイル編集で承認時にサンドボックスをバイパス可能に
- web_fetch がアクティブなサンドボックスネットワークポリシーに従い、ホストが sandbox.allowBypass でオプトインした場合に一度限りのバイパスを承認可能に
- Chronicle およびセッション SQL で正確なローカルアシスタント使用量を表示
- 別のリポジトリからリモートセッションを再開する前に確認を表示
- 読み取り専用リモートセッション作成を最初のメッセージ送信まで遅延
- CLI フッターに推論努力度ラベルを表示
- /settings でサンドボックス userPolicy 設定の説明を表示
- /copy を Wayland 環境で wl-copy にフォールバック
- /sandbox add-path サジェスチョンをリクエスト時のみ表示するよう改善
- /chronicle cost-tips をより精度の高いエビデンスベースの推奨に改善
- Alt（macOS では Option）を押しながらスクロールで1行ずつ移動
- 大規模セッションの再開と切り替えを高速化
- 大きな差分での /diff レンダリングとスクロールを高速化
- タイムラインが満杯時も /model ピッカー全体を画面に保持し、検索ボックスとヒントバーがクリップされないよう再スナップ
- Windows で CLI がヘルパープロセスを起動する際の一時コンソールウィンドウを非表示に
- 自動承認タイムラインエントリにリクエスト対象（コマンド、パス、URL、ツール名など）を含める
- /allow-all auto が AUTO_APPROVAL 環境変数やフィーチャーフラグではなく実験モード（/experimental on または --experimental）を必要とするよう変更
- /rubber-duck をプレ認証ヘルプおよびセルフドキュメンテーションに表示
- /diff ローカル編集に新しい未追跡ディレクトリ内のファイルを含める
- 画面より高いプロンプトがピン留めヘッダーに折り畳まれずスクロール可能に
- --stream オフ時にプロンプトモードで完全なアシスタントレスポンスを表示
- スタッフ専用コマンドをプレ認証ヘルプとセルフドキュメンテーションから非表示に
- `skill list` でスキル読み込みエラーと警告を表示
- CLI モデルピッカーでモデルの閲覧とフィルタリングを容易に
- タイムライン再構築時の二次的な処理を除去しセッション再開を高速化
- 静的コンテキストがプロンプトバジェットの大部分を使用する際に警告し、会話余地が少ない場合にリクエストをブロック
- セッションデータベースの読み書き時の CLI レスポンシブネスを改善
- スラッシュコマンドおよびテーマピッカーのオプション説明を改善
- 不活性なサンドボックスホストリストと「終了時にポリシーをクリア」トグルを削除
- バックグラウンドセッションのワーキングディレクトリ変更時にセッション分割ビューのブランチラベルを更新
- 既にロード済みのセッションに戻る際の不要な MCP リロードをスキップ
- 大規模モノレポで tgrep インデクサーのメモリ不足を防止し、インデクサーが kill された場合は繰り返し再起動せず ripgrep にフォールバック

### バグ修正

- `copilot init` が非対話モードでハングする問題を修正
- 空の --session-id= 値を無視せず拒否するよう修正
- 空の --resume= 値で新しいセッションを開始せず拒否するよう修正
- トークンタイミングに小数ミリ秒が含まれる場合のセッション再開を維持
- [DIR] で始まるファイル名の @メンション補完に対応
- ピン留めプロンプトのフレームをスクロール中に正しく配置
- macOS でクリップボード読み書き時の pasteboard stderr を抑制
- レンダリングされた Markdown 出力でコロンを保持
- 複数のプラグインが同じ MCP サーバーを定義する際に警告を表示
- 大きなファイルの切り詰めやマルチラインシークレットの欠落なしにデバッグログを収集
- ローカルブランチ名が PR ヘッドリファレンスと異なるワークツリーブランチから PR を検出
- バイパスモードがポリシーで無効な場合にオートパイロット権限プロンプトをスキップ
- WSL UNC パスをローカルファイルで許可
- CLI 起動時に --reasoning-effort を確実に適用
- タブ展開ターミナルでスラッシュコマンドピッカー行を維持
- ツールコール ID に句読点が含まれる場合のモデル切り替えを維持
- Esc ダブルプレスで実行中のメインターンを中断（キューメッセージをフラッシュ）、またはメインエージェントがアイドル時にバックグラウンドエージェントを停止
- Windows で CLI 終了時の良性クラッシュレポートを防止
- tmux パススルーでクリップボード書き込みをラップし外部ターミナルにコピーを送信
- OAuth ゲート MCP サーバーが再接続後にツールを登録するよう修正
- ワンショットセッションで CLI 認証を維持
- Ctrl+C で ask_user およびエリシテーションプロンプトを Escape と同様に閉じる
- キャンセルされた MCP OAuth リクエストに Authenticate を表示
- /help で rewind/undo が2回リストされる問題を修正
- OAuth キャンセル後に MCP サーバーを needs-auth 状態に維持
- 切り詰められたタイムラインツール引数で絵文字を保持
- /terminal-setup 出力でプレーンな Shift+Enter を表示
- ACP authenticate が成功を返す前に Copilot ログインを必須に
- リポスコープのプラグイン MCP サーバーをリポ無効化時またはセッション離脱時に切断
