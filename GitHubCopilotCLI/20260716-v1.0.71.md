# GitHub Copilot CLI v1.0.71 - 2026-07-16 アップデート情報

## リリース日

2026-07-16

## GitHub Copilot CLI バージョン

1.0.71

## 概要

プランモードでのワークスペース変更ブロック、/worktree と /move コマンドの分離、プラグインマーケットプレイスコマンドの追加、サブエージェントネスト深度のデフォルト引き下げなど、安全性と操作性を強化する多数の新機能・改善・バグ修正を含むアップデートです。

## 変更点

### 新機能

- プランモードでビルトインツールコールによるワークスペース変更をハードブロック（ファイル編集や変更を伴うシェルコマンドをブロック。MCP や外部ツールは引き続き許可）
- /worktree と /move を分離: /worktree は新しいワークツリーを作成しコミットされていない変更を残し、/move がそれらを新しいワークツリーに移動
- `plugins marketplace` サブコマンド（list、add、remove）を追加してプラグインマーケットプレイスを管理
- plugins marketplace browse および update コマンドを追加
- /voice devices コマンドを追加し、ボイスモード用マイクの選択と永続化が可能に
- CLI でエクステンション駆動のインタラクション用キャンバスサポートを追加
- Ctrl+R コマンド履歴の設定可能な最大値を追加
- /settings ダッシュボードに Repo および Repo (local) スコープタブを追加
- ピン留めプロンプト設定を /settings に追加
- GitHub MCP ツールセット/ツール設定を settings.json 経由で永続化（githubMcpToolsets、githubMcpTools など）
- サイドバーセッションを再起動後も永続化
- オートパイロットモードをタスク完了後も維持する `stayInAutopilot` 設定（デフォルト false）を追加
- `copilot update` および `/update` で `stable` チャンネルを受け付け可能に
- ? を2回押してクイックヘルプを閉じ、リテラル ? でプロンプトを開始可能に
- シェル補完でポジショナル引数の選択肢を提案
- ctrl+x → x でセッションを閉じ、ctrl+x → h で分割サイドバーを非表示に

### 改善

- `copilot -p --autopilot` でバックグラウンドシェルやエージェントがターンを超えて存続してもハングしなくなり、COPILOT_TASK_WAIT_TIMEOUT_SECONDS タイムアウトを通常の `-p` と同様に尊重
- /subagents モデルピッカーを再度開いた際に各エージェントの推論努力度とコンテキスト層を保持
- 長時間セッションで30分後にメモリコンテキストをリフレッシュ
- サーバー変更時に MCP ツールリストを最新に維持
- 終了後に長時間実行のバックグラウンド git プロセスを残さないよう改善
- 起動時に無効な settings.json がある場合、問題のある値を特定する警告を表示（設定を無視する代わりに）
- /terminal-setup が実際の kitty キーボードサポートのないターミナルでセットアップをスキップしなくなった
- タスクおよびサブエージェントで使用可能なビルトインエージェントを制限
- サンドボックスファイルシステムポリシーを LSP ファイル読み取りおよびリネーム編集に適用
- マーケットプレイスメタデータで空の owner および author メールを許可
- 短いターミナルですべての MCP Server Type オプションを表示
- `copilot skill list` およびその JSON 出力で無効化されたスキルをマーク
- /chronicle cost-tips の推奨をリッチなコストプロファイルで改善
- Markdown 内のスタンドアロン hex カラーをインラインでハイライト
- /chronicle cost-tips にローカルおよびクラウドコストプロファイルを追加
- ターン途中でオートパイロットに切り替えた際、同ターン中に質問された内容に自動回答
- エイリアスでシェルツールを要求するカスタムエージェントが対応する read、list、stop シェルツールも受信
- スラッシュコマンドとオートコンプリートが大文字小文字に関係なくマッチ（例: /SESSION が /session と同様に動作）
- /plugin list にリポジトリ有効化されたプラグインを表示
- Linux で /app 起動メッセージとダウンロードリンクを即時表示
- --max-autopilot-continues が NaN、負数、小数値を拒否するよう検証
- NO_COLOR が chalk でカラーレベルがキャッシュされていても CLI で尊重
- /settings 変更後にセッションオプション（シェルフラグ、ストリーミング、カスタムエージェントデフォルト）を即時適用
- /model のフォーカス行をスクリーンリーダーにアナウンス
- フォーカスされたピッカー行をスクリーンリーダーにアナウンス
- /agent で選択済みカスタムエージェントを一度だけ表示し、ファイル名と表示名が異なる場合にソースラベルを維持
- /model の価格バナーをモデルが一致しない場合にクリア
- /share file session および /share html session がフルセッションセレクターを出力パスとして使用しないよう修正
- --context を新しいインタラクティブセッションで尊重
- デフォルトのサブエージェントネスト深度の最大値を6から4に引き下げ（使用量ベース課金ユーザーは `subagents.maxDepth` で最大128まで調整可能）
- ターゲット検証コマンドと軽量インストールガイダンスをデフォルトで使用
- bare `copilot mcp` および `copilot skill` がヘルプを表示して終了コード 0 を返すよう変更（`copilot plugin` と統一）
- リポジトリ有効化プラグインの MCP サーバーをリポジトリが無効化した場合やセッションが宣言リポジトリを離れた場合に切断

### バグ修正

- モデルピッカーで検索結果が空の場合に非表示モデルの推論努力度やコンテキストウィンドウが変更される問題を修正し、空の状態で表示される不活性キーヒントを非表示に
- プラグインルートスキルを /plugin:plugin ではなく /plugin として表示
- 設定ファイル内の1つのフック項目が不正な場合でも有効なフックを維持
- write(path) の拒否が指定パスのみをブロックするよう修正
- --add-github-mcp-tool "\*" ですべての GitHub MCP ツールを有効化
- 空の未追跡ファイルをファントム追加行なしにレンダリング
- copilot skill add 失敗時にクリーンな失敗メッセージを表示
- 空白の設定配列アイテムで Enter を押した際にエラーを表示（空の値を保存する代わりに）
- /settings でデフォルトが登録されたブール値を Enter 1回でトグル
- /cwd でフォルダー信頼を拒否した際にライブセッションを開いたまま前のフォルダーに戻る
- ワークスペース MCP 設定が不正またはロードできない場合に警告を表示
- -p モードで不正な allowed_models.txt ポリシーエラーをクリーンに表示
- 同期済みセッションを名前で再開する際の偽の複数一致エラーを修正
- 既存セッションに --session-id と --name を使用した際にエラーを表示
- --plugin-dir プラグインを copilot plugin list に表示
- 切り替え時にバックグラウンドセッションを維持
- -p --stream off 出力で bare #number GitHub 参照をリンク
- スタートアップバナーを once 設定時に初回起動のみ表示
- --plugin-dir の警告をターミナルに表示
- 不正なカスタムエージェントの実際のロードエラーを表示
- --continue と --resume の併用を拒否
- プロンプトモードで --share または --share-gist のエクスポート失敗時に非ゼロで終了
- サーバーモードでキャッシュ済みトークンから OAuth MCP サーバーに再接続
- マーケットプレイスプラグインインストール時に保存済み Git クレデンシャルヘルパーを利用可能に維持
- セッションをプロンプト間、再起動間、ワークスペースツール間で作業ディレクトリに紐付け
- ask_user 選択プロンプトで常にカスタム回答を提供
- 隠しファイルを作成するカスタムエージェント名を拒否
- 不正な --allow-tool および --deny-tool パターンをエラーメッセージ付きで拒否
- /tasks のシェル詳細で完了タスクの保持されたシェル出力を表示
- プラグインコマンド失敗時の重複 Error: プレフィックスを削除
- シェル補完がサブコマンドをフラグ値として提案しないよう修正
- /usage アクティビティグラフで単数形のメッセージカウントを表示
- /cd がファイルやアクセス不能なディレクトリへの切り替えを防止
- ? でクイックヘルプオーバーレイを閉じた際に迷子の ? がプロンプトに残らないよう修正
- --sandbox および --no-sandbox がサンドボックス機能不使用時にインタラクティブ起動中に「無視」警告を表示（以前は非インタラクティブモードでのみ表示）
- /mcp サーバー詳細ビューでラッパーだけでなく引数付きの完全なコマンドを表示
- 空の /lsp ログ（LSP Services）パネルの不活性 navigate および view-log ヒントを非表示に
- プロンプトがブロックされた場合に非インタラクティブプロンプト実行を失敗コードで終了
- リデザインされたインラインモデルピッカーで Auto ディスカウントを表示
- 新しいセッションがアクティブセッションの cwd ではなくデフォルトディレクトリで開始
- Fish 補完がクローズドチョイスフラグに対してのみ enum 値を提供
- インタラクティブシェルコマンドがシェルマネージャ破棄後に「unknown shellManager handle」エラーの代わりにリトライ可能な「reconfiguring」メッセージで失敗するよう修正。またシェルコンテキスト再構成中の読み取り時にデタッチコマンドの完了通知が失われなくなった
- /model ピッカーで Auto モデルの説明をクリック可能な Learn More リンク付き Markdown として表示
- クイックヘルプオーバーレイを ? で閉じた後に迷子の ? がプロンプトに残らない問題を修正
