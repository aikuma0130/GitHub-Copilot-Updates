# GitHub Copilot CLI v1.0.72 - 2026-07-20 アップデート情報

## リリース日

2026-07-20

## GitHub Copilot CLI バージョン

1.0.72

## 概要

agentStop フックの無限ループ防止、OS サンドボックス内での git/gh 認証のオプトイン対応、/plugins コマンドの大幅な機能拡充、Sessions サイドバーのキーボード・マウス操作対応など、安全性・操作性・プラグイン管理を強化する多数の新機能・改善・バグ修正を含むアップデートです。

## 変更点

### 新機能

- OS サンドボックス内での git および gh 認証をオプトインで利用可能に
- `/plugins` に `update`/`uninstall` 動詞を追加し、`enable`/`disable`/`remove` で `--plugin`/`--mcp`/`--skill` フラグまたは位置引数でプラグイン・MCP サーバー・スキルを操作可能に。`/plugins install --skill` でスキルのインストールにも対応
- `/plugins help` コマンドとスキル・MCP・マーケットプレイス管理機能を追加し、/plugin との完全な互換性を実現
- プロンプトで `$` を入力すると現在のセッションディレクトリでインタラクティブシェルを起動（`/settings shellShortcut on` で有効化、デフォルトはオフ）
- CLI から `copilot plugins install --skill <file, URL, or directory>` でスキルをインストール可能に（`--scope project` でリポジトリへのインストール対応）
- `/model --session` (`-s`) で現在のセッションのみモデル・推論エフォート・コンテキストウィンドウを変更可能に（グローバル設定は変更なし）
- Sessions サイドバーがキーボードとマウスで操作可能に（矢印キーで選択移動、Enter またはクリックでセッション切替、`n` で新規セッション作成、`x` 2回でセッション閉じ）
- `--plugin`、`--mcp`、`--skill` フラグをプラグイン変更操作に追加
- `copilot plugins remove --skill` でスキルの削除をサポート
- マルチターンサブエージェントを常時有効化し、実行中のエージェントにフォローアップメッセージ送信可能に
- Claude Haiku 4.5+ でツール検索を有効化
- インラインコードで記述された16進カラーコード（例: `#FF0000`）をカラースウォッチとして表示。`renderHexColors` 設定（デフォルトオン）で切替可能
- スケジュールされたプロンプトをエージェントがビジーの時にステアリングメッセージとして配信

### 改善

- `agentStop` フックが常にブロックする場合の無限ループを防止: 8回連続ブロック後にターンを終了し、`stop_hook_active` フラグでフック側が強制継続を検知可能に
- サンドボックスの macOS キーチェーンアクセスをデフォルトでオフに変更し、隔離を強化（必要な場合は /sandbox で再有効化）
- ライフサイクルおよびサブエージェントフックコマンドが /cd 後の現在のセッションディレクトリで実行されるように
- `/mcp delete` で MCP サーバーを削除すると、実行中のバックグラウンドプロセスも停止
- `/sandbox` の切替時にローカル MCP サーバーのみ再起動し、リモートサーバーは接続を維持
- コマンド承認が /cd でリポジトリを切り替えた後に引き継がれなくなった
- `/terminal-setup` で VS Code、Cursor、Windsurf を親プロセスから検出
- `copilot skill list` がスキル名と説明からターミナル制御文字を除去し、細工されたスキルによる ANSI エスケープシーケンスの注入を防止
- `/settings` でデフォルト値を表示し、ブール値をデフォルトに戻せるように
- マネージド設定が要求する場合、リモートコントロールに SSO を要求
- `/settings show` の出力でシークレット値をマスク
- プラン承認メニューをモデル間で決定論的に
- `/add-dir` ディレクトリをターン間でエージェントコンテキストに維持
- split-pane チャットビューで ask-user およびエリシテーション入力を折り返し表示
- コンパクト編集行の展開時にフルファイルパスを表示

### バグ修正

- GitHub タブの「Open in web」アクションが Windows で確実にブラウザを起動するように修正
- Ctrl+X /model でモデル変更時に貼り付けたプロンプト内容が保持されるように修正
- ファイル作成時に末尾改行を含むファイルで追加行数が1行多く表示される問題を修正
- `/worktree` と `/move` で多数の類似名ブランチや残存ワークツリーディレクトリがある場合に自動生成ブランチ名でのワークツリー作成が失敗する問題を修正（数値サフィックスの検索上限5を撤廃し、古い未登録ワークツリーディレクトリをスキップ）
- `/worktree <task>` がメインリポジトリで間欠的にキックオフタスクを実行する問題を修正
- `/worktree` と `/move` がソースが既に信頼済みの場合に新しいワークツリーに切替前にフォルダ信頼を伝播するように修正（不要なフォルダ信頼プロンプトを回避）。/move が git stash を正確にアドレスし、ワークツリー作成が残存ディレクトリをスキップ
- リネームされたセッションでメッセージがなくワークスペースがキャッシュスナップショットのみの場合に終了再開ヒントを表示
- 遅い MCP サーバーが最終的に接続した際に接続メッセージを表示
- セッションエクスポートでインラインコードおよびトップレベルのフェンスドコードブロック内の山括弧を保持
- `constructor` や `__proto__` のような名前の MCP サーバーのステータスを正しく表示
- セッション閉じ時にセッションハイライトを最も近いライブ行に維持（Sessions split view とスタンドアロン Sessions タブの両方）
- Markdown リスト内のインライン16進カラースウォッチのパディングを修正
- バッファード出力（`-p --stream off` および詳細画面）でネストされた Markdown リストが正しくレンダリングされるように修正（サブバレットが親アイテム行に結合・フラット化されなくなり、親の下にインデント）
- 修飾 vim キー（Ctrl+K、大文字 J/K）がツール許可プロンプトやテキスト入力セレクトメニューで選択を移動しないように修正
- `/terminal-setup` が JSON 構文エラーを含む VS Code keybindings.json の書き換えを拒否するように修正
- 絵文字ショートコード（例: :tada:）が印刷出力や PR/issue/gist 出力で余分な末尾スペース付きでレンダリングされる問題を修正
