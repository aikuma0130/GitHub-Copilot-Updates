# GitHub Copilot CLI v1.0.74 - 2026-07-23 アップデート情報

## リリース日

2026-07-23

## GitHub Copilot CLI バージョン

1.0.74

## 概要

Open Plugin Spec v1プラグインマニフェストとmcp.json設定のサポート追加、プランモード用モデル選択コマンドの追加、およびIDE連携やサブエージェントタイムラインに関する複数のバグ修正が含まれるリリース。

## 変更点

### 新機能
- Open Plugin Spec v1 プラグインマニフェストおよび mcp.json 設定のサポートを追加
- `/model plan`（または `/model --plan`）コマンドを追加し、プランモード中に使用するモデルを選択可能に（モデルID、`off` でクリア、またはIDなしでピッカーを表示。プランモード終了時にセッションモデルに戻る）
- 初回起動時にデフォルトサンドボックスへのオプトインスプラッシュを表示
- gemini-3.6-flash のサポートを追加

### 改善
- サブエージェントタイムラインで、プロンプトがメインエージェントからか別のサブエージェントからかを識別可能に
- セッション多重化時に、あるセッションの開いたダイアログが別のセッションに漏れなくなり、切り替え時に該当ピッカーが再表示される
- `$` インタラクティブシェルショートカットがエージェント動作中でもシェルを開けるように改善
- skill の disable-model-invocation フラグを完全に尊重するように改善
- 参加中の言語サーバーがリクエストされたものと異なるシンボルを報告した場合に警告を表示
- ステアリングがシェル出力待機を中断しても、実行中のコマンドを停止しないように改善
- Responses リクエストのサイズ制限を引き上げ
- プランモードでセッションフォルダー内のプランニングアーティファクトを許可しつつ、セッションフォルダー外のファイル変更はブロックするように改善
- レジューム検索がセッションタイトルの空白の違いに関わらずマッチするように改善

### バグ修正
- `/search` バーが開いている状態で `?` を入力すると、クイックヘルプを開く代わりにテキストとして入力されるように修正
- IDE連携がCLIのMCPサーバーリロードやディレクトリ変更時に確実に再接続するように修正
- マルチターンサブエージェントタイムラインが `/tasks` 再表示後にすべてのプロンプトとレスポンスを正しい順序で表示するように修正
- `/mcp add` および `/mcp edit` ウィザードで環境変数値の `=` 文字（base64パディングなど）が保持されるように修正
- リモートセッションアップロードが Mission Control の 400/404 永続エラーに対してリトライを停止するように修正
- /settings フッターに Tab でスコープタブを切り替える表示を追加
- ツール結果の画像がオーバーサイズの場合にダウンスケールし、CAPI Responses リクエストが継続するように修正
