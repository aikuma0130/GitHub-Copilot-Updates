# VS Code の Auto モデル選択がタスクに基づいたルーティングに対応

## 日付

2026-05-20

## ステータス

GA

## 概要

VS Code の Copilot Auto モデル選択が、リアルタイムのモデル可用性・健全性メトリクスとタスク特性に基づいて最適なモデルを自動ルーティングするように改善された。有料サブスクライバーは 10% の乗数割引を受けられる。

## 詳細

### 動作の仕組み

Auto はリアルタイムのモデル可用性と信頼性シグナルを評価し、以下の観点でタスクを分析して最適なモデルを選択する：

- 推論の複雑さ
- コード生成の難易度
- バグ診断の困難さ
- ツールオーケストレーションの必要性

### 主な特徴

- **透明性**: モデルレスポンスにホバーすると使用モデルを確認可能
- **制御**: Auto と特定モデルをいつでも切り替え可能
- **ポリシー遵守**: 管理者が設定したモデルポリシーを尊重
- **多様なモデルアクセス**: サブスクリプションとポリシーに応じて複数モデルファミリーを活用

### プレミアムリクエスト使用量

Auto は選択したモデルに基づいて課金され、現在は 0x〜1x 乗数のモデルに限定。有料サブスクライバーは Auto 使用時に 10% の乗数割引が適用される（例: 1x モデル使用時は 0.9 プレミアムリクエスト）。

### 効率性

Auto は自然なキャッシュ境界に沿ってルーティングし、不要なキャッシュ関連コストを回避。高推論・高トークンモデルが不要なタスクには効率的なモデルを使用することで、品質を損なわずにトークン効率を向上。
