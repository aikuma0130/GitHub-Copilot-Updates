# GitHub Issues のエージェント自動化コントロールがパブリックプレビュー

## 日付

2026-07-23

## ステータス

Public Preview

## 概要

GitHub Issues でエージェントが提案するラベル・タイプ・担当者変更・クローズなどのアクションに対し、承認・確信度・根拠の表示による制御機能が追加。自動化と人間の監視のバランスを実現。

## 詳細

### 主な機能

- **承認ワークフロー**: エージェントが提案する変更を自動適用せず、Issue 上のパネルに保留して手動レビュー・承認が可能
- **確信度レベル**: 各アクションに高・中・低の確信度スコアが付与
  - 高確信度: 自動適用
  - 中・低確信度: 提案として保留し、レビューを促す
- **根拠の記録**: すべてのアクション（自動適用・提案問わず）に理由が記録され、監査証跡として機能
- **しきい値の設定**: リポジトリ管理者が確信度のしきい値を設定し、どのレベルから手動レビューを要求するか制御可能
- **検索フィルター**: `has:suggestions` で保留中の提案がある Issue を検索可能

### 対応アクション

- ラベルの変更
- フィールドの更新
- Issue タイプの変更
- Issue のクローズ
- 担当者のアサイン

### 対応プラットフォーム

- GitHub Agentic Workflows
- Copilot クラウドエージェントの自動化
- API 経由のアクセス

### 設計思想

承認機能はワークフロー支援ツールとして設計されており、セキュリティメカニズムではない。権限を持つエージェントは設定により承認をバイパス可能。
