# Visual Studio Code v1.109.0（January 2026）— GitHub Copilot 関連アップデート

## リリース日

2026-02-04

## VS Code バージョン

1.109.0（リカバリリリース: 1.109.1 セキュリティ修正、1.109.2 チャット問題の修正、1.109.3 メッセージステアリング・エージェントフック・Claude 互換性・スキルをスラッシュコマンドとして使用、1.109.4 バグ修正、1.109.5 バックグラウンドエージェントのスラッシュコマンド・セッション名変更・Kitty キーボードサポート）

## 概要

VS Code v1.109.0（2026年1月リリース）では、GitHub Copilot のエージェント機能が大幅に強化されました。メッセージステアリングやエージェントフック、Plan エージェントなどの新機能に加え、Claude 互換性の実現、Agent Skills の一般提供（GA）、MCP アプリのサポートなど、AI アシスタントとしての機能がさらに進化しています。また、チャット UI の改善、ターミナルサンドボックス、自動承認ルールなど、日常の開発体験を向上させる多数の変更が含まれています。

---

## GitHub Copilot 関連の変更点

### メッセージステアリングとキューイング (Experimental)

`chat.requestQueuing.enabled` 設定により、リクエストの実行中でもフォローアップメッセージを送信できるようになりました。メッセージ送信時に3つの選択肢が表示されます。「Add to Queue」はメッセージをキューに追加し、現在のリクエスト完了後に順次処理します。「Steer with Message」は実行中のリクエストにメッセージを追加して方向を修正します。「Stop and Send」は現在のリクエストを停止して新しいメッセージを送信します。この機能により、エージェントとの対話がよりスムーズになり、待ち時間を有効に活用できます。v1.109.3 以降で利用可能です。

---

### Anthropic モデルでのシンキングトークン表示

Claude モデル使用時に、モデルの思考プロセス（シンキングトークン）がチャットレスポンスに表示されるようになりました。関連する設定として、`chat.thinking.style` で思考トークンの表示スタイルを制御でき、`chat.agent.thinking.collapsedTools` でツール使用時の思考表示を折りたたむかどうかを設定できます。`chat.agent.thinking.terminalTools` はターミナル関連ツールでの思考表示を制御し、`chat.tools.autoExpandFailures` はツール実行が失敗した場合に自動的に詳細を展開するかどうかを設定します。これにより、モデルがどのように推論しているかを把握しやすくなります。

---

### チャットレスポンスでの Mermaid ダイアグラム

チャットが `renderMermaidDiagram` ツールを使用して、インタラクティブな Mermaid ダイアグラムをレンダリングできるようになりました。レンダリングされたダイアグラムはパン（移動）やズーム操作に対応しており、クリックで拡大表示も可能です。さらに、エディタでダイアグラムを開いたり、Mermaid のソースコードをコピーしたりすることもできます。アーキテクチャ図やフローチャート、シーケンス図などを会話の中で視覚的に確認できるため、設計や説明がより直感的になります。

---

### Ask Questions ツール (Experimental)

`chat.askQuestions.enabled` 設定を有効にすると、エージェントがユーザーに対して明確化のための質問を行えるようになります。単一選択や複数選択のオプション、フリーテキスト入力などの形式で質問が表示されます。これにより、エージェントは曖昧な指示に対して確認を取りながら作業を進められるため、より正確で意図に沿った結果を生成できます。

---

### Plan エージェント

`/plan` コマンドにより、4つのフェーズで構成されるワークフローを使って高品質な実装計画を作成できるようになりました。最初の「Discovery（発見）」フェーズではコードベースを調査し、「Alignment（整合）」フェーズでは要件の確認と方針のすり合わせを行います。「Design（設計）」フェーズで具体的な実装設計を策定し、最後の「Refinement（洗練）」フェーズで計画を精査して最終化します。この段階的なアプローチにより、複雑なタスクに対してもより整理された高品質な実装計画が得られます。

---

### コンテキストウィンドウの詳細表示

チャット入力エリアにコンテキストウィンドウインジケータが追加されました。ホバーすると、カテゴリごとのトークン使用量の内訳が表示されます。これにより、現在の会話でどの程度のコンテキストが使用されているか、またどのカテゴリ（システムプロンプト、ユーザーメッセージ、ツール結果など）がトークンを消費しているかを一目で把握できます。コンテキストウィンドウの残量を意識しながら効率的に会話を進めることが可能になります。

---

### インライン チャット UX の刷新 (Preview)

インラインチャットのユーザー体験が大幅に刷新されました。`inlineChat.affordance` 設定により、インラインチャットをトリガーするための新しいアフォーダンス（UI 要素）が利用可能になりました。また、`inlineChat.renderMode` 設定では、軽量でコンテキストに応じたレンダリングモードが導入され、コードの流れを妨げることなくインラインで提案を確認できます。エディタ内での AI 支援がよりシームレスになり、コーディング中の作業を中断せずに Copilot を活用できます。

---

### モデルピッカーのモデル説明表示

モデルピッカーでモデルにホバーまたはキーボードフォーカスを当てると、モデルの詳細情報が一覧で表示されるようになりました。各モデルの特徴や強みを一目で確認でき、タスクに適したモデルを素早く選択できます。

---

### ターミナルコマンド出力の改善

ターミナルでのコマンド出力が大幅に改善されました。インラインで実行される Node.js、Python、Ruby のコードに対してシンタックスハイライトが適用されるようになりました。また、作業ディレクトリの表示、コマンドの意図の説明表示、出力のストリーミング表示、インタラクティブな入力サポートが追加されています。さらに、非表示のターミナルをすべて削除する機能も追加され、ターミナル管理がより効率的になりました。

---

## エージェント機能の強化

### エージェントセッション管理

新しいセッションタイプピッカーが導入され、ローカルエージェント、バックグラウンドエージェント、クラウドエージェントの切り替えが容易になりました。Agent Sessions ビューも改善され、セッションの管理がより直感的になっています。`chat.agentsControl.enabled` 設定を有効にすると、コマンドセンターにエージェントのステータスインジケータが表示され、現在のエージェントの状態を常に把握できます。

---

### サブエージェント

サブエージェントが並列で実行できるようになりました。`github.copilot.chat.searchSubagent.enabled` 設定で有効になる検索サブエージェントは、分離されたエージェントループ内で実行され、メインのエージェントの処理をブロックしません。また、`chat.customAgentInSubagent.enabled` 設定により、カスタムエージェントをサブエージェントとして利用することも可能になりました。これにより、複雑なタスクを複数のエージェントで分担して効率的に処理できます。

---

### クラウドエージェント

クラウドエージェントにモデル選択機能が追加され、使用するモデルを指定できるようになりました。Claude や Codex などのサードパーティコーディングエージェントとの連携もサポートされています。カスタムエージェントのサポートやマルチルートワークスペースの改善も含まれており、チェックアウト操作が常に表示されるようになりました。

---

### バックグラウンドエージェント

バックグラウンドエージェントでカスタムエージェント、画像添付、マルチルートワークスペースがサポートされました。各ターンの終了時に自動コミットが行われるようになり、作業の進捗が自動的に保存されます。v1.109.5 ではスラッシュコマンドのサポートとセッション名の変更機能が追加されました。

---

### エージェントセッション ウェルカムページ (Experimental)

`workbench.startupEditor` 設定で `agentSessionsWelcomePage` を指定すると、新しいウェルカムページが表示され、エージェントセッションの一覧が確認できます。これにより、VS Code 起動時にすぐに進行中のエージェントセッションにアクセスでき、前回の作業をスムーズに再開できます。

---

### エージェントフック (Preview)

`chat.hooks.enabled` 設定により、エージェントのライフサイクルの各ポイントでカスタムシェルコマンドを実行できるようになりました。8つのフックイベントが用意されており、`PreToolUse`（ツール使用前）、`PostToolUse`（ツール使用後）、`SessionStart`（セッション開始時）、`Stop`（停止時）などが含まれます。フック形式は Claude Code および Copilot CLI の形式と互換性があり、既存のワークフローをそのまま活用できます。v1.109.3 以降で利用可能です。

---

## カスタマイズと設定

### スキルをスラッシュコマンドとして使用

Agent Skills がプロンプトファイルと並んでスラッシュコマンドとして利用可能になりました。フロントマターで `user-invocable` プロパティを設定するとユーザーが直接呼び出せるスラッシュコマンドとして登録され、`disable-model-invocation` プロパティを設定するとモデルによる自動呼び出しを無効にできます。これにより、スキルの呼び出し方法をきめ細かく制御できます。

---

### /init でワークスペースを AI 用にセットアップ

`/init` スラッシュコマンドにより、コードベースの分析に基づいてワークスペースインストラクションを生成または更新できるようになりました。プロジェクトの構造、使用技術、コーディング規約などを自動的に検出し、AI がより適切な支援を提供するためのインストラクションファイルを作成します。

---

### Agent Skills が一般提供（GA）

Agent Skills が一般提供（GA）となり、デフォルトで有効になりました。`Chat: Configure Skills` コマンドからスキルの管理が可能です。デフォルトのスキルファイルの配置場所は `.github/skills` および `.claude/skills` ディレクトリです。スキルを使用することで、エージェントに特定のタスクやワークフローに関する専門知識を付与できます。

---

### 組織全体のカスタムインストラクション

`github.copilot.chat.organizationInstructions.enabled` 設定により、組織レベルのカスタムインストラクションが自動的に適用されるようになりました。管理者が設定した組織全体のインストラクションがすべてのメンバーに対して反映されるため、チーム全体で一貫したコーディング基準や規約を維持できます。

---

### カスタムエージェントの呼び出し制御

カスタムエージェントのフロントマターに `user-invocable`、`disable-model-invocation`、`agents` 属性が追加されました。`user-invocable` はユーザーによる直接呼び出しの可否を制御し、`disable-model-invocation` はモデルによる自動呼び出しを無効にします。`agents` 属性では、そのカスタムエージェントが利用できるサブエージェントを指定できます。

---

### カスタムエージェントの複数モデルサポート

カスタムエージェントのフロントマターで複数のモデルを指定でき、フォールバック機能が利用可能になりました。優先順位の高いモデルが利用できない場合、次のモデルに自動的に切り替わるため、可用性が向上します。

---

### チャットカスタマイズ診断

新しい診断ビューが追加され、読み込まれたすべてのカスタマイズファイル（インストラクション、プロンプト、スキル、カスタムエージェントなど）とそのステータスを一覧で確認できるようになりました。カスタマイズが意図した通りに適用されているかの確認やトラブルシューティングに役立ちます。

---

### 言語モデルエディタの改善

言語モデルの設定が強化されました。プロバイダーごとに複数の設定を保持できるようになり、Azure モデルの設定もサポートされました。プロバイダーグループの管理機能が追加され、`chatLanguageModels.json` ファイルを通じてモデル設定を一元管理できます。

---

### 言語モデル設定

Plan 実装時のデフォルトモデル、インラインチャットのデフォルトモデル、エージェントハンドオフ時の model パラメータなど、用途に応じたモデルの指定が可能になりました。タスクの種類に応じて最適なモデルを使い分けることで、より効率的な開発が実現できます。

---

### エージェントカスタマイズスキル (Experimental)

カスタマイズに関する専門知識を持つ組み込みスキルが追加されました。このスキルはエージェント、インストラクション、プロンプト、スキルのカスタマイズ方法についてエージェントに教示し、ユーザーがカスタマイズを設定する際のガイダンスを提供します。

---

## Claude 統合

### Claude 互換性

VS Code が Claude の設定ファイル（`CLAUDE.md`、`.claude/rules`、`.claude/settings.json`）を直接読み込めるようになりました。フックの形式も互換性があるため、Claude Code 向けに作成された設定やワークフローを VS Code でもそのまま利用できます。v1.109.3 以降で利用可能です。

---

### Claude エージェント (Preview)

GitHub Copilot サブスクリプションを通じて Claude Agent SDK の統合が利用可能になりました。エージェントフック、スキル、カスタムエージェント、MCP サーバーなどの VS Code の機能と連携して動作します。Claude の強力な推論能力を活用しながら、VS Code のエコシステムの恩恵を受けることができます。

---

## MCP と外部連携

### MCP アプリ (Experimental)

MCP サーバーがチャット内でインタラクティブな HTML コンテンツをレンダリングできるようになりました。これにより、MCP サーバーが提供するツールの結果をリッチな UI で表示でき、データの可視化やインタラクティブなフォームなどをチャット内に直接埋め込むことが可能です。

---

### 外部コード検索インデックス

実験的な機能として、外部インデックスを使用した高速コード検索がサポートされました。大規模なコードベースでも、事前に構築されたインデックスを活用することで、コード検索の速度と精度が大幅に向上します。

---

## Copilot メモリ

### Copilot メモリ

Copilot がセッション間でユーザーの好みや設定を記憶できるようになりました。メモリパネルを通じてメモリの管理（確認、編集、削除）が可能です。一度伝えた好み（コーディングスタイル、使用フレームワーク、命名規則など）を毎回繰り返す必要がなくなり、よりパーソナライズされた支援を受けられます。

---

## セキュリティと承認

### ターミナルサンドボックス

ターミナルでのコマンド実行環境を制限できるサンドボックス機能が追加されました。macOS ではサンドボックス、Linux では Landlock、Windows ではポリシーベースの制限がそれぞれサポートされています。エージェントが実行するコマンドの影響範囲を制限することで、意図しないシステム変更を防ぎ、セキュリティを強化できます。

---

### 自動承認ルール

特定のツールやコマンドに対してきめ細かい自動承認ルールを設定できるようになりました。`/autoApprove` スラッシュコマンドで自動承認を設定し、`/disableAutoApprove` スラッシュコマンドで無効化できます。頻繁に使用する安全なツールやコマンドを自動承認に設定することで、確認ダイアログの回数を減らし、作業効率を向上させることができます。

---

## その他の変更

### Edit モードの非表示化

Edit モードがエージェントドロップダウンからデフォルトで非表示になりました。`chat.editMode.hidden` 設定でこの動作を変更できます。エージェントモードの普及に伴い、Edit モードの利用頻度が低下したことを反映した変更です。
