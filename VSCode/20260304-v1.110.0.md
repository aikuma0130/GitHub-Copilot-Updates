# Visual Studio Code v1.110.0（2026年2月リリース） — GitHub Copilot 関連アップデート

## リリース日

2026-03-04

## VS Code バージョン

1.110.0（リカバリーリリース: 1.110.1 — コアおよび GitHub Copilot Chat 拡張機能のセキュリティアップデートを含む）

## 概要

VS Code v1.110.0（2026年2月リリース）では、GitHub Copilot に関連する多数の新機能と改善が導入されました。バックグラウンドエージェントの改善、Claude エージェントの大幅なパフォーマンス向上、エージェントプラグインの実験的サポート、ブラウザツールの統合、NES（次の編集提案）の強化、チャットセッションのフォーク機能など、開発者体験を大きく向上させる変更が含まれています。また、アクセシビリティの改善や、カスタマイズ性の向上にも重点が置かれています。

---

## GitHub Copilot 関連の変更点

### バックグラウンドエージェントの改善

バックグラウンドエージェントに複数の重要な改善が加えられました。`/compact` スラッシュコマンドによるコンテキストコンパクション（会話の要約・圧縮）がサポートされ、長時間のセッションでもコンテキストウィンドウを効率的に管理できるようになりました。また、プロンプトファイル、フック、スキルに対応したスラッシュコマンドが追加され、バックグラウンドエージェントのカスタマイズ性が向上しています。さらに、バックグラウンドエージェントセッションの名前変更が可能になり、複数セッションの管理がより直感的になりました。

---

### Claude エージェントの改善

Claude エージェントに大幅な改善が加えられ、パフォーマンスが大きく向上しました。ステアリング（指示の誘導）とキューイング（リクエストの順序管理）がサポートされ、複数のリクエストをより効率的に処理できるようになりました。セッションの名前変更機能、コンパクション付きのコンテキストウィンドウレンダリングが追加されています。スラッシュコマンドとして `/compact`、`/agents`、`/hooks` が利用可能になり、`getDiagnostics` ツールも追加されました。これらの変更により、Claude エージェントの全体的なパフォーマンスが大幅に改善されています。

---

### Agent Debug パネル (Preview)

新しい Agent Debug パネルがプレビュー機能として導入されました。このパネルでは、チャットイベント、カスタマイズイベント、システムプロンプト、ツール呼び出しをリアルタイムで確認できます。従来の Diagnostics チャットアクションに代わるもので、より詳細なデバッグ情報を提供します。チャートビューによる視覚的な階層表示にも対応しており、エージェントの動作を直感的に把握できます。コマンドパレットまたはチャットビューの歯車アイコンから開くことができます。

---

### 自動承認のスラッシュコマンド

チャット入力から直接、グローバルな自動承認設定を切り替えるためのスラッシュコマンドが追加されました。`/autoApprove` で自動承認を有効にし、`/disableAutoApprove` で無効にできます。それぞれ `/yolo` と `/disableYolo` というエイリアスも用意されており、素早く入力できます。これにより、エージェントの操作承認フローをチャットインターフェースから直接制御でき、作業の流れを中断せずに設定を変更できるようになりました。

---

### Edit モードと Ask モードの変更

Edit モードがデフォルトで非表示になりました（`chat.editMode.hidden` 設定で制御）。Ask モードはカスタムエージェント定義によってバックエンドが実装されるようになり、アーキテクチャが刷新されました。ユーザーは Edit エージェントの宣言を確認でき、これをカスタムエージェント作成の出発点として利用することが可能です。この変更により、モードの概念がよりシンプルになり、エージェントベースのカスタマイズが容易になりました。

---

### Ask Questions ツールの改善

Ask Questions ツール（ユーザーへの質問機能）が VS Code コアに統合され、信頼性が大幅に向上しました。サブエージェントを含むあらゆるコンテキストで一貫して動作するようになっています。保留中の質問を閉じることなくステアリングメッセージを送信できるようになり、操作の柔軟性が高まりました。また、⌥N（Option+N）と ⌥P（Option+P）によるキーボードナビゲーションがサポートされ、質問間の移動がスムーズになりました。

---

### チャット中の自動サスペンド防止

チャットリクエストの処理中に、VS Code が OS の自動サスペンド（スリープ）を防止するようになりました。これにより、長時間のエージェントタスクやコード生成リクエストの実行中にシステムがスリープ状態に入ることを防ぎ、処理の中断を回避できます。チャットリクエストが完了すると、通常のサスペンド動作に戻ります。

---

### エージェントプラグイン (Experimental)

`chat.plugins.enabled` 設定で有効化できる実験的なエージェントプラグイン機能が導入されました。拡張機能ビューから、スキル、ツール、フックをパッケージ化したバンドルをインストールできます。`@agentPlugins` で検索することで、利用可能なプラグインを見つけることができます。デフォルトのマーケットプレイスとして `copilot-plugins` と `awesome-copilot` が設定されており、`chat.plugins.marketplaces` 設定でカスタムマーケットプレイスを追加することも可能です。これにより、エージェントの機能を柔軟に拡張できるエコシステムが構築されつつあります。

---

### エージェント向けブラウザツール (Experimental)

`workbench.browser.enableChatTools` 設定で有効化できる実験的なブラウザツールが導入されました。エージェントが統合ブラウザを使用して Web ページと対話できるようになります。利用可能なツールには、`openBrowserPage`（ページを開く）、`navigatePage`（ページ遷移）、`readPage`（ページ読み取り）、`screenshotPage`（スクリーンショット取得）、`clickElement`（要素クリック）、`hoverElement`（要素ホバー）、`dragElement`（要素ドラッグ）、`typeInPage`（ページ内テキスト入力）、`handleDialog`（ダイアログ処理）、`runPlaywrightCode`（Playwright コード実行）が含まれます。ページはプライベートなインメモリセッションで実行されるため、既存のブラウザセッションに影響を与えません。

---

### チャットからエージェントカスタマイズを作成

チャットインターフェースから直接、各種エージェントカスタマイズを作成するためのスラッシュコマンドが追加されました。`/create-prompt`（プロンプト作成）、`/create-instruction`（インストラクション作成）、`/create-skill`（スキル作成）、`/create-agent`（エージェント作成）、`/create-hook`（フック作成）が利用可能です。進行中の会話からパターンを抽出してカスタマイズを作成することもでき、反復的なワークフローの効率化に役立ちます。クイックピックメニューからもアクセスできます。

---

### usages と rename ツール

`usages` ツールが更新され、新しい `rename` ツールが追加されました。これらのツールは既存の拡張機能や LSP（Language Server Protocol）の機能を再利用することで、高精度なリファクタリングを実現します。シンボルの使用箇所の検索やリネーム操作が、言語サーバーの知識を活用してより正確に行われるようになり、エージェントによるコード変更の品質が向上しました。

---

### プランのセッションメモリ

エージェントが作成したプラン（計画）がセッションメモリに永続化されるようになりました。プランは会話のターン間で利用可能であり、関連のないメッセージの後でも呼び出すことができます。コンテキストコンパクション後もアクセス可能なため、長時間のセッションでもプランの情報が失われることがありません。これにより、複雑なタスクの計画と実行がより一貫して行われるようになりました。

---

### コンテキストコンパクション

会話の自動および手動コンパクション（圧縮・要約）機能が導入されました。`/compact` スラッシュコマンドで手動コンパクションを実行でき、コンテキストウィンドウが一杯になった場合は自動的にコンパクションが行われます。ローカルセッション、バックグラウンドセッション、Claude セッションのすべてで利用可能です。カスタムインストラクションもサポートされており、例えば `/compact focus on the database schema decisions`（データベーススキーマの決定に焦点を当てて圧縮）のように、コンパクション時に重視する内容を指定できます。

---

### Explore サブエージェントによるコードベース検索

Plan エージェントが調査タスクを Explore サブエージェントに委譲する機能が追加されました。Explore サブエージェントは読み取り専用であり、検索ツールとファイル読み取りツールのみを使用します。Claude Haiku 4.5 や Gemini 3 Flash などの高速モデル上で動作するため、コードベースの調査を迅速に行うことができます。`chat.exploreAgent.defaultModel` 設定で使用するモデルを変更することも可能です。この仕組みにより、プランニングフェーズでのコードベース理解が効率化されました。

---

### インラインチャットとチャットセッション

エージェントセッションが既にファイルを変更している場合、同じファイルでのインラインチャットは独立した変更を行うのではなく、既存のセッションにキューイングされるようになりました。これにより、エージェントセッションとインラインチャットの変更が競合することがなくなり、一貫性のあるコード変更が保証されます。ファイル単位での変更の整合性が向上しました。

---

### チャットセッションのフォーク

`/fork` スラッシュコマンドで、現在の会話履歴をすべて含む新しいセッションを作成できるようになりました。任意のチャットリクエストにホバーすることで、そのチェックポイントからフォークすることも可能です。フォークされたセッションは元のセッションから独立しており、異なるアプローチを試したり、特定の時点から別の方向に会話を進めたりする際に便利です。

---

### モデルピッカーの再設計

モデル選択用のドロップダウンが再設計されました。新しいデザインでは、Auto（自動選択）、おすすめ・最近使用したモデル、その他のモデルのセクションに分かれており、検索ボックスも搭載されています。モデルにホバーすると詳細情報がリッチホバーで表示されるため、各モデルの特徴を確認しながら適切なモデルを選択できるようになりました。

---

### コンテキスト付きチップスでの機能紹介 (Experimental)

`chat.tips.enabled` 設定で有効化できる実験的な機能です。チャットビューにコンテキストに応じたチップス（ヒント）が表示され、GitHub Copilot の機能をより効果的に発見できるようになります。チップスはユーザーの利用パターンに合わせてカスタマイズされるため、個々のユーザーにとって最も関連性の高い機能が紹介されます。

---

### カスタムシンキングフレーズ

`chat.agent.thinking.phrases` 設定で、エージェントの推論中やツール呼び出し中に表示されるローディングテキストをカスタマイズできるようになりました。デフォルトのフレーズを完全に置き換えるモードと、既存のフレーズに追加するモードの両方がサポートされています。チームやプロジェクトに合わせた独自のメッセージを設定することで、開発体験をパーソナライズできます。

---

### ターミナルツール呼び出しの折りたたみ

`chat.tools.terminal.simpleCollapsible` 設定により、ターミナルの出力が折りたたみ可能なセクションとして表示されるようになりました。長いターミナル出力がチャットの可読性を低下させることなく、必要に応じて展開して詳細を確認できます。これにより、エージェントがターミナルコマンドを多用するタスクでも、チャットインターフェースが整理された状態を保てます。

---

### チャット応答と確認のOS通知

`chat.notifyWindowOnResponseReceived` と `chat.notifyWindowOnConfirmation` 設定により、チャット応答や確認が必要な場合に OS レベルの通知を受け取れるようになりました。従来はウィンドウがフォーカスされていない時のみ通知が送られていましたが、`"always"` に設定することで、VS Code にフォーカスがある場合でも通知を受け取ることが可能になりました。長時間のエージェントタスク中に他の作業をしている場合に特に便利です。

---

### インラインチャット ホバーモード

`inlineChat.renderMode` 設定で新しいホバーベースの UI が選択可能になりました。このモードでは、インラインチャットの進捗状況と結果がエディタの右上隅に表示されます。従来のインラインウィジェットと比較して、編集中のコードの表示を妨げにくく、よりコンパクトなインターフェースを提供します。

---

### インラインチャット アフォーダンス

`inlineChat.affordance` 設定により、インラインチャットをより簡単に開始する方法が追加されました。テキスト選択時にメニューが表示され、そこからインラインチャットを起動できます。エディタ内の選択範囲の横、またはガターに表示されるオプションから利用可能です。インラインチャットの発見性と操作性が向上し、より直感的にアクセスできるようになりました。

---

### アクセシビリティの改善

アクセシビリティに関する多くの改善が行われました。アクセシブルビューでのシンキングコンテンツの表示切替、質問カルーセルのアクセシビリティ向上、質問・確認の通知機能が追加されています。TODO リストにフォーカスするためのキーバインド、カーソル位置の保持、検索・フィルターのアクセシビリティヘルプの改善も含まれます。さらに、クイック入力のスクリーンリーダー対応の改善、ステアリングインジケーター、アクセシビリティスキル、チャット内のチェックマーク表示など、包括的なアクセシビリティの強化が実施されました。

---

### NES（次の編集提案）の改善

NES（Next Edit Suggestions）が大幅に強化されました。長距離 NES が導入され、ファイル内のどこにでも編集を予測できるようになりました。従来はカーソル付近の編集提案に限定されていましたが、ファイル全体を対象とした予測が可能になっています。また、ステータスバーに NES の積極度（eagerness）オプションが追加され、提案の頻度や範囲をユーザーの好みに合わせて調整できるようになりました。

---

### AI共著者トレーラー

`git.addAICoAuthor` 設定により、AI が生成したコードを含むコミットに `Co-authored-by` トレーラーを自動的に追加する機能が導入されました。設定オプションとして `off`（無効）、`chatAndAgent`（チャットおよびエージェントモードで生成されたコードのみ）、`all`（すべての AI 生成コードを含む）が選択可能です。これにより、AI との協調作業の透明性が向上し、コミット履歴から AI の関与を明確に追跡できるようになりました。
