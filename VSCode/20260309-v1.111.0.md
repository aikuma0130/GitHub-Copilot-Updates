# Visual Studio Code v1.111.0 (March 2026) — GitHub Copilot 関連アップデート

## リリース日

2026-03-09

## VS Code バージョン

1.111.0（初回週次 Stable リリース）

## 概要

VS Code v1.111.0 では、Autopilot モードの導入やエージェントスコープフック、デバッグイベントスナップショットなど、GitHub Copilot のエージェント機能を中心に大幅な機能強化が行われました。チャット体験の改善やターミナル統合の実験的機能も追加されています。

---

## GitHub Copilot 関連の変更点

### Autopilot とエージェント権限

チャットビューに新しい権限ピッカーが導入され、エージェントの動作を 3 段階で制御できるようになりました。

- **Default Approvals（デフォルト承認）**: 構成済みの設定に基づいてツール呼び出しやアクションの承認を行います。従来どおりのワークフローで、ユーザーが個別に承認・拒否を判断します。
- **Bypass Approvals（承認バイパス）**: すべてのツール呼び出しとアクションを自動承認します。エラーが発生した場合は自動的にリトライを行い、手動介入なしで処理を継続します。
- **Autopilot（プレビュー）**: すべてのツール呼び出しを自動承認するだけでなく、エージェントからの応答にも自動で返答し、タスクが完了するまで処理を継続します。エージェントが `task_complete` を報告するまで自律的に動作するため、完全なハンズフリー体験を実現します。

この機能は `chat.autopilot.enabled` 設定で有効化でき、Insiders 版ではデフォルトで有効になっています。

---

### エージェントスコープフック (Preview)

`chat.useCustomAgentHooks` 設定を有効にすることで、カスタムエージェントの `.agent.md` ファイルの YAML フロントマターにエージェントスコープのフックを定義できるようになりました。

エージェントスコープフックは、特定のエージェントが選択された場合、または `runSubagent` 経由で呼び出された場合にのみ実行されます。これにより、エージェントごとに固有の前処理・後処理ロジックを設定でき、カスタムエージェントの柔軟性と制御性が大幅に向上します。

---

### デバッグイベントスナップショット

チャットで `#debugEventsSnapshot` を使用して、エージェントのデバッグイベントのスナップショットをコンテキストとして添付できるようになりました。

Agent Debug パネルのスパークルアイコンをクリックすると、現在のデバッグイベントをチャットへの添付ファイルとして追加できます。添付されたスナップショットを開くと、対応するタイムスタンプでフィルタリングされたパネルが表示され、エージェントの動作をより効率的にデバッグ・分析できます。

---

### チャットチップの改善

チャットチップ（ヒント表示）が再設計され、構造化されたオンボーディングジャーニーが導入されました。

- 最初に基礎的なチップが表示され、その後に生産性向上のためのチップが順次表示されます。
- チャットセッションが 1 つだけ表示されている場合にのみチップが表示されるため、作業の邪魔になりません。
- キーボードショートカットが含まれており、素早い操作方法を学べます。
- `/init` や `/fork` などのスラッシュコマンドに関するチップも追加され、新規ユーザーがこれらの便利な機能を発見しやすくなりました。

---

### AI CLI プロファイルグループ (Experimental)

`terminal.integrated.experimental.aiProfileGrouping` 設定を有効にすると、AI CLI ターミナルプロファイル（例: GitHub Copilot CLI）がターミナルプロファイルドロップダウンの上部に専用グループとして表示されるようになります。

これにより、AI 関連のターミナルツールへのアクセスが容易になり、通常のシェルプロファイルと AI CLI プロファイルを明確に区別できるようになります。この機能は実験的段階のため、今後のリリースで変更される可能性があります。
