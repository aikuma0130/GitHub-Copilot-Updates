# Visual Studio Code v1.112.0 (March 2026) — GitHub Copilot 関連アップデート

## リリース日

2026-03-18

## VS Code バージョン

1.112.0

## 概要

VS Code v1.112.0 では、Copilot CLI の大幅な機能強化、エージェントのデバッグ・トラブルシューティング機能の追加、MCP サーバーのサンドボックス化やプラグイン管理の改善など、GitHub Copilot 関連の多数のアップデートが含まれています。

---

## GitHub Copilot 関連の変更点

### Copilot CLI でのメッセージステアリングとキューイング

Copilot CLI セッションで、前のリクエストがまだ実行中であっても新しいメッセージを送信できるようになりました。これにより、エージェントの動作中にリアルタイムで指示を修正（ステアリング）したり、次に実行してほしいタスクをキューに入れたりすることが可能です。長時間のタスク実行中でも、方向転換や追加指示をスムーズに行えるため、作業効率が大幅に向上します。

---

### Copilot CLI への委任時の変更プレビュー

コミットされていない変更がある状態で Copilot CLI にタスクを委任する際、Chat ビューに保留中の変更一覧が表示されるようになりました。ユーザーはワークツリー内の変更に対して、以下の操作を選択できます。

- **コピー**: 変更を CLI セッションにコピーする
- **移動**: 変更を CLI セッションに移動する
- **無視**: 変更を無視して CLI セッションを開始する

これにより、未コミットの変更が意図せず失われるリスクを防ぎつつ、柔軟にエージェントへ作業を引き継ぐことができます。

---

### Copilot CLI ターミナル出力でのクリック可能なファイルリンク

設定: `github.copilot.chat.cli.terminalLinks.enabled`

Copilot CLI のターミナル出力に表示されるファイルパスが、クリック可能なリンクとして認識されるようになりました。ターミナルリンク検出は `~/.copilot/session-state/` 配下のパスを認識し、以下の形式のパスに対応しています。

- 絶対パス（例: `/home/user/project/file.ts`）
- チルダ付きパス（例: `~/project/file.ts`）
- 相対パス（例: `./src/file.ts`）

これにより、CLI 出力からファイルをすばやくエディタで開くことができ、デバッグや確認作業が効率化されます。

---

### Copilot CLI の権限レベル

設定: `chat.autopilot.enabled`

Copilot CLI セッションに対して、以下の3段階の権限レベルが設定できるようになりました。

- **Default Permissions（デフォルト権限）**: 標準的な権限設定で、ツール実行時に承認を求めます
- **Bypass Approvals（承認スキップ）**: ツール実行時の承認プロンプトをスキップします
- **Autopilot（自動操縦）**: 完全自律モードで、エージェントがすべての操作を自動的に実行します

Autopilot モードは VS Code Insiders ではデフォルトで有効になっています。権限レベルを適切に設定することで、セキュリティと利便性のバランスを調整できます。

---

### /troubleshoot でエージェント動作のトラブルシューティング (Preview)

設定: `github.copilot.chat.agentDebugLog.enabled`, `github.copilot.chat.agentDebugLog.fileLogging.enabled`

新しい `/troubleshoot` スキルにより、会話内のエージェントデバッグログを分析できるようになりました。このスキルは以下の問題の診断に役立ちます。

- ツールやサブエージェントの使用状況の把握
- インストラクションの読み込み状況の確認
- レスポンスが遅い原因の特定
- ネットワーク関連の問題の調査

エージェントの動作が期待通りでない場合に、この機能を使って原因を素早く特定できます。プレビュー機能のため、設定で有効化する必要があります。

---

### エージェントデバッグログのエクスポートとインポート (Preview)

設定: `github.copilot.chat.agentDebugLog.enabled`

エージェントセッションのデバッグログをエクスポートおよびインポートできるようになりました。主な用途は以下の通りです。

- 他のチームメンバーとログを共有して問題を共同で調査する
- ログをオフラインで分析する
- 過去のセッションの動作を振り返る

50 MB を超えるファイルの場合は警告が表示されます。大規模なセッションのログは容量が大きくなる可能性があるため、必要に応じてログの範囲を絞ることを推奨します。

---

### エージェントの画像・バイナリファイルサポート

設定: `chat.imageCarousel.enabled`, `imageCarousel.explorerContextMenu.enabled`

エージェントがディスク上の画像ファイルやバイナリファイルをネイティブに読み取れるようになりました。

- **画像ファイル**: エージェントやツールの出力に含まれる画像をチャット内で選択でき、画像カルーセルビューで開くことができます
- **バイナリファイル**: バイナリファイルは16進ダンプ形式で表示され、内容を確認できます

この機能により、スクリーンショットやデザインファイルを参照しながらの開発作業や、バイナリデータの検証がエージェントとの対話内でシームレスに行えるようになります。

---

### 自動シンボル参照

シンボル（クラス名、関数名、メソッド名など）をコピーしてチャットに貼り付けると、VS Code が自動的にシンボル参照（`#sym:Name` 形式）として貼り付けるようになりました。

シンボル参照として貼り付けることで、Copilot はそのシンボルの定義や使用箇所などのコンテキストを自動的に取得し、より正確な回答を生成できます。プレーンテキストとして貼り付けたい場合は、`Ctrl+Shift+V`（テキストとして貼り付け）を使用してください。

---

### 親リポジトリでのカスタマイズ検出

設定: `chat.useCustomizationsInParentRepositories`

モノレポ構成において、VS Code がリポジトリルートまでの親フォルダからカスタマイズファイルを自動的に検出するようになりました。以下のすべてのカスタマイズファイルタイプに対応しています。

- `copilot-instructions.md`
- `AGENTS.md`
- `CLAUDE.md`
- インストラクションファイル
- プロンプトファイル
- カスタムエージェント
- スキル
- フック

モノレポのサブパッケージ内で作業している場合でも、ルートレベルで定義されたカスタマイズが自動的に適用されるため、設定の一貫性が保たれます。

---

### ローカル MCP サーバーのサンドボックス化

設定: `mcp.json` 内の `sandboxEnabled: true`

ローカル MCP サーバーをサンドボックス環境で実行できるようになりました。サンドボックス化されたサーバーは、ファイルシステムとネットワークアクセスが制限されます。

- サーバーが追加のアクセス権を必要とする場合、VS Code がユーザーに許可を求めるプロンプトを表示します
- macOS および Linux で利用可能です

この機能により、信頼性が不明な MCP サーバーを安全に実行でき、ファイルシステムやネットワークへの不正アクセスを防止できます。

---

### MCP エリシテーション UI の改善

MCP のエリシテーション（情報取得）フォームが、Ask Questions ツールと同じ UI を使用するように改善されました。これにより、ユーザーへの質問や入力要求の表示が統一され、一貫性のある体験が提供されます。異なる MCP サーバーからの問い合わせであっても、同じ見た目と操作感で対応できるようになります。

---

### プラグインと MCP サーバーの有効/無効切り替え

プラグインや MCP サーバーをアンインストールせずに有効/無効を切り替えられるようになりました。

- **グローバル設定**: すべてのワークスペースに対して有効/無効を設定
- **ワークスペース単位**: 特定のワークスペースでのみ有効/無効を設定

切り替えは以下の方法で行えます。

- MCP / プラグインの設定ページから操作
- Extensions / Customizations ビューで右クリックメニューから操作

一時的にプラグインや MCP サーバーを無効にしたい場合に、再インストールの手間なく簡単に管理できます。

---

### プラグインの自動更新

設定: `extensions.autoUpdate`

プラグインが自動的に更新されるようになりました。これにより、常に最新バージョンのプラグインを利用できます。

- npm および pypi から提供されるプラグインの場合、更新時にユーザーの承認が必要です
- それ以外のプラグインは、設定が有効であれば自動的に更新されます

セキュリティの観点から、パッケージマネージャー経由のプラグインは明示的な承認フローを経ることで、意図しない更新によるリスクを軽減しています。
