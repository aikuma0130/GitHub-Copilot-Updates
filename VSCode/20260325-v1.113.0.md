# Visual Studio Code v1.113.0 (2026年3月) — GitHub Copilot 関連アップデート

## リリース日

2026-03-25

## VS Code バージョン

1.113.0

## 概要

VS Code v1.113.0 では、Copilot CLI および Claude エージェントにおける MCP サポートやセッションフォークの導入、チャットカスタマイゼーションの一元管理 UI、ネストされたサブエージェントの解禁など、GitHub Copilot のエージェント機能とカスタマイズ性が大幅に強化されました。あわせて、モデルピッカーでのシンキングエフォート設定やプラグインマーケットプレイスの管理機能など、日常のワークフローを改善する多数の機能が追加されています。

---

## GitHub Copilot 関連の変更点

### Copilot CLI & Claude エージェントでの MCP サポート

VS Code で設定された MCP（Model Context Protocol）サーバーが、Copilot CLI および Claude エージェントにブリッジされるようになりました。これにより、ユーザーが定義した MCP サーバーだけでなく、ワークスペースの `mcp.json` ファイルで構成されたワークスペースサーバーも、CLI エージェントから直接利用できるようになります。VS Code 内でのみ使用可能だった MCP ツールが CLI セッションでもシームレスに活用でき、エージェントの能力が大きく拡張されます。

---

### Copilot CLI & Claude エージェントでのセッションフォーク

新しい設定 `github.copilot.chat.cli.forkSessions.enabled` により、Copilot CLI および Claude エージェントのセッションフォーク機能が利用可能になりました。会話履歴の任意の時点で既存セッションのコピーを作成でき、元のコンテキストを失うことなく、異なるアプローチを自由に探索できます。たとえば、あるプロンプトに対して複数の解決策を並行して試したい場合に、会話を分岐させて比較検討することが可能です。

---

### Copilot CLI & Claude CLI セッションのエージェントデバッグログ (Preview)

Copilot CLI および Claude エージェントセッション向けに、エージェントデバッグログパネルが利用可能になりました（プレビュー機能）。このパネルでは、エージェントのインタラクションを時系列で表示するイベントログを確認できます。これまではローカルセッションのみでサポートされていたデバッグログが、CLI セッションにも拡張されたことで、リモートやターミナルベースのエージェント操作のデバッグ・トラブルシューティングが容易になります。

---

### SDK API による Claude セッションリスト

VS Code が Claude エージェントのセッションおよびメッセージの一覧表示に、公式の Claude エージェント SDK API を採用するようになりました。以前はディスク上の Claude JSONL ファイルを直接パースしていたため、同期の問題が発生するリスクがありました。この変更により、セッションやメッセージが欠落する問題が解消され、セッション管理の信頼性と安定性が向上しています。

---

### ネストされたサブエージェント

新しい設定 `chat.subagents.allowInvocationsFromSubagents` により、サブエージェントが他のサブエージェントを呼び出すことが可能になりました。これにより、複雑な多段階ワークフローをエージェント間の連携で実行できるようになります。以前は無限再帰を防止するためにサブエージェントからの呼び出しが制限されていましたが、この設定を有効にすることで、より高度なタスクの自動化が実現可能です。

---

### プラグインマーケットプレイスの管理

新しいコマンド「**Chat: Manage Plugin Marketplaces**」が追加されました。このコマンドを使用すると、設定済みのすべてのプラグインマーケットプレイスを一覧表示できます。各マーケットプレイスのプラグインの閲覧、ローカルディレクトリの表示、マーケットプレイスの削除などの操作が可能です。複数のマーケットプレイスを利用している場合の管理が大幅に簡素化されます。

---

### プラグインインストール用 URL ハンドラー

URL を通じたプラグインマーケットプレイスおよびプラグインのインストールが可能になりました。以下の URL スキームに対応しています。

- **マーケットプレイスの追加**: `vscode://chat-plugin/add-marketplace?ref=<source>`
- **プラグインのインストール**: `vscode://chat-plugin/install?source=<source>`

`source` パラメーターには、GitHub のリポジトリ（`owner/repo` 形式）または Base64 エンコードされた Git URI を指定できます。これにより、ドキュメントや README からワンクリックでプラグインを導入できるようになります。

---

### Chat Customizations エディタ (Preview)

チャットカスタマイゼーションの作成と管理を一元的に行える新しい UI がプレビュー機能として追加されました。以下のタブで構成されており、各種カスタマイズを統合的に管理できます。

- **カスタムインストラクション** — エージェントへの指示をカスタマイズ
- **プロンプトファイル** — 再利用可能なプロンプトテンプレートの管理
- **カスタムエージェント** — 独自エージェントの定義と設定
- **エージェントスキル** — エージェントが利用可能なスキルの管理

エディタには構文ハイライトとバリデーション付きのコードエディタが組み込まれており、プロジェクトコンテキストからの AI による自動生成機能も備えています。また、エディタ内から MCP サーバーやエージェントプラグインを直接ブラウズすることも可能です。

---

### モデルピッカーでのシンキングエフォート設定

Claude Sonnet 4.6 や GPT-5.4 などの推論モデルにおいて、モデルピッカーにシンキングエフォート（思考量）のサブメニューが表示されるようになりました。設定可能なレベルは以下の 3 段階です。

- **Low** — 高速な応答を優先
- **Medium** — バランスの取れた推論
- **High** — より深い推論を実行

選択したエフォートレベルはモデルごとに保持され、会話をまたいで維持されます。モデルピッカーのラベルには現在選択されているエフォートレベルが表示されます。なお、この機能の導入に伴い、従来の設定 `github.copilot.chat.anthropic.thinking.effort` および `github.copilot.chat.responsesApiReasoningEffort` は非推奨となりました。

---

### チャット添付ファイルの画像プレビュー

新しい設定 `imageCarousel.chat.enabled` および `imageCarousel.explorerContextMenu.enabled` により、チャットに添付された画像のプレビュー機能が追加されました。任意の画像添付ファイルを選択すると、フル画像ビューアーが開きます。

主な機能:

- **ナビゲーション** — 矢印キー、キーボードショートカット、サムネイルストリップによる画像間移動
- **セクション分類** — 会話ターンごとに画像がグループ化して表示
- **ズーム＆パン** — クリック、Option+Click（macOS）/ Ctrl+Click（Windows/Linux）、スクロール / ピンチ操作による拡大・縮小と移動

また、エクスプローラーのコンテキストメニューからも画像ファイルに対してこの画像ビューアーを利用できます。
