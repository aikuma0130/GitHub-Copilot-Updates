# Visual Studio Code v1.114.0 (2026年4月) — GitHub Copilot 関連アップデート

## リリース日

2026-04-01

## VS Code バージョン

1.114.0

## 概要

VS Code v1.114.0 では、チャット体験の効率化に焦点が当てられ、`#codebase` ツールのセマンティック検索への一本化とインデックス管理の簡素化、チャット応答のコピー機能改善、過去セッションのトラブルシューティング機能、画像カルーセルの動画対応など、GitHub Copilot のチャットワークフローを改善する複数の変更が含まれています。

---

## GitHub Copilot 関連の変更点

### チャット添付ファイルの画像カルーセルで動画プレビューに対応

**設定**: `imageCarousel.chat.enabled`、`imageCarousel.explorerContextMenu.enabled`

[v1.113 で導入された](https://code.visualstudio.com/updates/v1_113#_images-preview-for-chat-attachments)画像カルーセルが、動画にも対応しました。チャット添付ファイルやエクスプローラーのコンテキストメニューから、動画の再生とナビゲーションが可能になります。

主な機能:

- **動画再生** — コントロール付きの再生機能
- **ナビゲーション** — 矢印キーやサムネイルを使った画像・動画間の移動

---

### チャットの最終レスポンスをコピー

チャットビューには会話全体や特定のレスポンスをコピーするコマンドがすでに用意されていましたが、従来はエージェントの思考プロセスやツール呼び出しも含まれていました。

今回追加された **Copy Final Response** コマンドにより、エージェントの最終的な Markdown レスポンス（すべてのツール呼び出しが完了した後の最後のセクション）のみをコピーできるようになりました。チャットのコンテキストメニューからアクセスでき、レスポンスの共有がより簡単になります。

---

### ワークスペース検索の簡素化

Copilot がコードベースのセマンティック検索を行うための `#codebase` ツールが大幅に簡素化されました。

#### 主な変更点

- **セマンティック検索に特化**: `#codebase` は純粋にセマンティック検索のみに使用されるようになりました。以前はフォールバックとして精度の低いファジーテキスト検索が使われることがありましたが、今後は `#codebase` はセマンティック検索に限定されます。エージェントは必要に応じて別途テキスト検索やファジー検索を実行できます。

- **インデックス管理の統一**: 以前は「ローカルインデックス」と「リモートインデックス」が別々に管理されていましたが、現在は「コードベースがセマンティックにインデックスされているかどうか」という単一の状態に統一されました。裏側ではインデックスの一部がローカルに、一部がリモートに保存される場合がありますが、ユーザーが自分でインデックスを管理する必要はなくなりました。

#### ユーザーへの影響

- `#codebase` ツールは常にセマンティックで、一貫した結果を提供します
- Copilot がセマンティック検索が有効な場面で自動的に `#codebase` を使用し、インデックスもオンデマンドで自動構築されます
- 以前インデックス済みと表示されていたワークスペースは、ローカルの非セマンティックインデックスを使用していた場合、再インデックスが必要になることがあります
- GitHub リポジトリを持たない非常に大きなコードベースは、現時点ではインデックスできない場合があります（サポートは段階的に拡大中）

ワークスペースがセマンティックにインデックスされていない場合でも、Copilot のテキスト検索、grep、シンボル検索などを通じて良好な結果を得ることができます。詳細は [ワークスペースガイド](https://code.visualstudio.com/docs/copilot/reference/workspace-context) を参照してください。

### 過去のチャットセッションのトラブルシューティング (Preview)

**設定**: `github.copilot.chat.agentDebugLog.enabled`、`github.copilot.chat.agentDebugLog.fileLogging.enabled`

`/troubleshoot` コマンドによるトラブルシューティング機能が拡張され、過去のチャットセッションを参照できるようになりました。これにより、問題を再現する必要なく、事後的に問題を調査できます。

使用方法:

1. `/troubleshoot` コマンドを入力
2. プロンプトに `#session` を含める
3. セッションピッカーから過去のセッションを選択

カスタムインストラクションが無視された理由やレスポンスが遅い原因など、エージェントの動作に関する問題を診断する際に活用できます。また、「**+**」（**Add Context**）> 「**Sessions**」からもセッションを添付できます。

### Claude エージェントを無効化するグループポリシー（エンタープライズ）

管理者がグループポリシーを使用して、チャットにおける Claude エージェント統合を無効化できるようになりました。このポリシーが適用されると、`github.copilot.chat.claudeAgent.enabled` 設定は組織によって管理され、ユーザーは Claude エージェントを有効にすることができなくなります。

ポリシーはブーリアン型で、キーは `Claude3PIntegration` です。詳細は [デバイス管理ポリシー](https://code.visualstudio.com/docs/setup/enterprise#_device-management) のドキュメントを参照してください。

---

### 細粒度のツール承認（Proposed API）

言語モデルツールの承認フローにおいて、特定の引数の組み合わせに対してスコープを限定した承認が可能になる API が提案されました。

たとえば、組み込みの「Run VS Code Command」ツールは任意の VS Code コマンドを実行できますが、ユーザーは `editor.action.formatDocument` は常に承認したいが、他のコマンドはその都度承認したいという場合があります。この API により、ツールの実装側が特定のコマンドに承認スコープを限定できるため、ユーザーは各コマンドを個別に承認することが可能になります。

詳細は [Fine grain tool approval](https://github.com/microsoft/vscode/blob/af50a47c13e23e0b3c46719dbd92fe00144362a5/src/vscode-dts/vscode.proposed.toolInvocationApproveCombination.d.ts) の API 提案を参照してください。

### Edit Mode の非推奨化スケジュール（継続）

VS Code v1.110 で正式に非推奨となった **Edit Mode** について、スケジュールの詳細が更新されました。設定 `chat.editMode.hidden` を使用して一時的に Edit Mode を再有効化することができますが、この設定は v1.125 までのサポートとなります。v1.125 以降、Edit Mode は完全に削除され、設定による有効化もできなくなります。
