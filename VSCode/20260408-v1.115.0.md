# Visual Studio Code v1.115.0 (2026年4月) — GitHub Copilot 関連アップデート

## リリース日

2026-04-08

## VS Code バージョン

1.115.0

## 概要

VS Code v1.115.0 では、エージェント中心の開発体験を強化する新しいコンパニオンアプリ「VS Code Agents」がプレビューとして導入され、統合ブラウザのエージェントツール改善やバックグラウンドターミナルへの入力・通知機能など、GitHub Copilot エージェントの操作性を向上させる複数の変更が含まれています。

---

## GitHub Copilot 関連の変更点

### Visual Studio Code Agents アプリ (Preview)

**Visual Studio Code Agents** は、VS Code Insiders と並行して提供される新しいプレビューコンパニオンアプリです。エージェント中心の開発に最適化されています。

主な機能:

- **プロジェクト間のタスク並列化** — 複数のリポジトリにまたがるエージェントセッションを並列に開始できます（各セッションは独自の worktree で分離されます）。コンテキストの素早い切り替えが可能で、人間とエージェントのレビューを反復できます。
- **モニタリングとレビュー** — セッションの進捗状況を追跡し、diff をインラインで表示し、エージェントへのフィードバックを残し、アプリ内からプルリクエストを作成できます。
- **カスタマイズの引き継ぎ** — カスタムインストラクション、プロンプトファイル、カスタムエージェント、MCP サーバー、フック、プラグインなどが Agents アプリでも動作します。テーマなどの他の VS Code カスタマイズも引き継がれます。
- **追加インストール不要** — VS Code Insiders と一緒に配布されます。OS のスタートメニューやアプリケーションフォルダから起動するか、コマンドパレットから **Chat: Open Agents Application** を実行します。

現在は VS Code Insiders でのみ利用可能な、急速に進化中のプレビュー機能です。

### 統合ブラウザ — エージェントツールの改善

**設定**: `workbench.browser.enableChatTools`

統合ブラウザのエージェントツールに複数の改善が加えられました。

#### より分かりやすいツールラベル

エージェントがブラウザツールを呼び出した際、ツールコールのラベルがより説明的になり、対象のブラウザタブへの直接リンクが表示されるようになりました。従来の「Clicked element in browser」のような汎用的な表示から、「Right-clicked header banner in Test Page」のように具体的な操作内容とページ名が表示されます。

#### 長時間実行スクリプトのサポート

`Run Playwright Code` ツールの長時間実行スクリプトのサポートが改善されました。デフォルトで5秒以上かかるスクリプトは、エージェントがポーリングできる遅延結果を返すようになりました。

#### 重複タブの削減

エージェントが繰り返しブラウザタブを開くことが抑制されるようになりました。エージェントが新しいタブを開こうとした際、同じホストに既にタブが開かれている場合は、エージェントが明示的なフラグを渡さない限り新しいタブは開かれません。

---

### ターミナルツールの改善 — バックグラウンドターミナルへの入力送信

従来、バックグラウンドターミナルは読み取り専用で、`get_terminal_output` のみが利用可能でした。フォアグラウンドターミナルがタイムアウトしてバックグラウンドに移動した場合、エージェントはそれ以上操作できなくなる制限がありました。

新しい `send_to_terminal` ツールにより、エージェントはバックグラウンドターミナルとの対話を継続できるようになりました。例えば、SSH セッションがパスワードプロンプトを待っている間にタイムアウトした場合でも、エージェントは必要な入力を送信して接続を完了できます。

---

### ターミナルツールの改善 — バックグラウンドターミナル通知 (Experimental)

**設定**: `chat.tools.terminal.backgroundNotifications`

従来、バックグラウンドでターミナルコマンドが実行中の場合、エージェントは `get_terminal_output` を手動で呼び出してステータスを確認する必要がありました。コマンドの完了や入力の必要性を知る手段がありませんでした。

新しい実験的な `chat.tools.terminal.backgroundNotifications` 設定により、バックグラウンドターミナルのコマンドが完了したり、ユーザー入力が必要になった際にエージェントに自動的に通知されるようになりました。フォアグラウンドターミナルがタイムアウトしてバックグラウンドに移動した場合にも適用されます。エージェントは通知を受けて、出力のレビューや `send_to_terminal` ツールによる入力の提供などの適切なアクションを取ることができます。

---

### Edit Mode の非推奨化スケジュール（継続）

VS Code v1.110 で正式に非推奨となった **Edit Mode** について、引き続きスケジュールが案内されています。設定 `chat.editMode.hidden` を使用して一時的に Edit Mode を再有効化することができますが、この設定は v1.125 までのサポートとなります。v1.125 以降、Edit Mode は完全に削除され、設定による有効化もできなくなります。
