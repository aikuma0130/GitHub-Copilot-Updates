# Visual Studio Code v1.116.0 (2026年4月) — GitHub Copilot 関連アップデート

## リリース日

2026-04-15

## VS Code バージョン

1.116.0

## 概要

VS Code v1.116.0 では、GitHub Copilot Chat が組み込み拡張機能となりインストール不要で利用可能になったほか、エージェントデバッグログの過去セッション参照、Copilot CLI での思考レベル設定、ターミナルツールのフォアグラウンド対応など、エージェント体験を中心に多数の改善が行われています。

---

## GitHub Copilot 関連の変更点

### GitHub Copilot が組み込み拡張機能に

GitHub Copilot Chat が VS Code の組み込み拡張機能（built-in extension）になりました。新規ユーザーは拡張機能をインストールすることなく、チャット、インライン提案、エージェントなどの Copilot 機能を利用できるようになります。

この変更は、VS Code をオープンソースの AI コードエディタにするための継続的な取り組みの一環です。Copilot を組み込み拡張機能として提供することで、新規ユーザーの導入障壁を下げ、初回起動時から AI 機能がシームレスに統合されます。

既存ユーザーには影響はなく、すでにインストール済みの Copilot 拡張機能はこれまで通り動作します。AI 機能を使用しない場合は、設定 `chat.disableAIFeatures` で無効化できます。

---

### エージェントデバッグログ — 過去セッションの参照

**設定**: `github.copilot.chat.agentDebugLog.fileLogging.enabled`

Agent Debug Log パネルでは、チャットセッション中のエージェント操作の時系列イベントログを表示し、プロンプト送信時の動作やチャットカスタマイズのデバッグに活用できます。

今回のリリースで、現在のセッションに加えて過去のセッションのログも参照できるようになりました。ログはローカルディスクに永続化され、セッション終了後でも過去のエージェント操作を確認・デバッグできます。

また、Agent Debug Logs パネルを有効化する設定が、トラブルシューティング設定 `github.copilot.chat.agentDebugLog.fileLogging.enabled` に統合されました。

---

### Copilot CLI での思考レベル（Thinking Effort）設定

ローカルエージェントセッションと同様に、Copilot CLI セッションでも言語モデルピッカーを使って推論モデルの思考レベル（thinking effort）を設定できるようになりました。思考レベルは、モデルが各リクエストにどの程度の推論を適用するかを制御し、応答品質とレイテンシのバランスを調整できます。

モデルピッカーで推論モデルを選択し、矢印をクリックすると利用可能な思考レベルが表示されます。利用可能なレベルはモデルによって異なり、推論モデル以外ではサブメニューは表示されません。

---

### チャットカスタマイズのウェルカムページ

**Chat: Open Customizations** コマンドまたは Chat ビューの歯車アイコンからアクセスできるチャットカスタマイズダイアログに、ウェルカムページが追加されました。すべてのエージェントカスタマイズの概要を一覧で確認できます。

ウェルカムページの **Customize Your Agent** 入力欄に自然言語で説明を入力すると、VS Code がエージェント、スキル、インストラクションなどのカスタマイズを自動的にドラフトしてくれます。

---

### ツール確認カルーセル (Experimental)

**設定**: `chat.tools.confirmationCarousel.enabled`

複数のツールコールの承認・拒否をより効率的に行うために、チャットにツール確認用のカルーセルコントロールが追加されました。カルーセルにより、会話をスクロールすることなく、コンパクトなナビゲーションで複数のツールコールを順番にレビュー・承認できます。

この機能は実験的で、設定 `chat.tools.confirmationCarousel.enabled` で制御されます。VS Code Insiders ではデフォルトで有効化されており、Stable 版ではフィードバックを収集しながら段階的にロールアウトされています。

---

### Visual Studio Code Agents アプリ (Preview) — アップデート

前回リリースで導入された **Visual Studio Code Agents** アプリに、以下のアップデートが行われました。

- **推論レベルの選択**: Copilot CLI セッションで推論モデルの思考レベルを設定可能に。
- **プランモードの自動起動**: CLI セッションで計画が必要な場合、プランモードが自動的に開始されるように。
- **Changes パネルの Files タブ**: Changes パネルで Files タブがデフォルト表示に。
- **セッション応答・テーマ・レンダリングの改善**: 応答処理、ビジュアルの一貫性、レンダリングパフォーマンスの改善。
- **アプリ名変更**: アプリ名が **Visual Studio Code Agents - Insiders** に変更。

VS Code のウェルカムページから「Try out the new Agents app」で直接起動できるエントリポイントも追加されました。

### ターミナルツール — フォアグラウンドターミナルのサポート

`send_to_terminal` および `get_terminal_output` エージェントツールが、エージェントが作成したバックグラウンドターミナルだけでなく、フォアグラウンドターミナルでも動作するようになりました。これにより、ターミナルパネルに表示されている任意のターミナル（実行中の REPL やインタラクティブスクリプトなど）からの出力読み取りや入力送信が可能になります。

---

### ターミナルツール — 入力体験の改善

ターミナル入力に関する複数の改善が行われました。

- **ターミナル入力の検出改善**: LLM ベースのプロンプト入力検出が削除されました。従来は各ターミナル出力チャンクごとに追加の LLM 呼び出しが発生し、レイテンシとトークン消費が増加していました。エージェントは `send_to_terminal` で直接入力を処理し、必要に応じて質問カルーセルでユーザーに確認を行います。
- **進捗メッセージの改善**: エージェントがターミナルに回答を送信する際、どの質問に対する回答かが表示されるようになりました（例: `Sending "my-project" to terminal (replying to: What is your project name?)`）。
- **Focus Terminal ボタン**: エージェントがターミナル入力を必要とする場合（パスワードプロンプトや `npm init` のようなインタラクティブインストーラーなど）、質問カルーセルに **Focus Terminal** ボタンが表示されるようになりました。ターミナルで直接入力を開始すると、カルーセルは自動的に閉じられ、ユーザーが直接入力を処理していることがエージェントに通知されます。

---

### ターミナルツール — バックグラウンドターミナル通知がデフォルト有効に

**設定**: `chat.tools.terminal.backgroundNotifications`

バックグラウンドターミナル通知がデフォルトで有効になりました。エージェントがバックグラウンドターミナルでコマンドを実行した際、コマンドの完了、タイムアウト、入力待ちの通知を自動的に受信します。これにより、エージェントはターミナル出力のポーリングを行うことなく、より迅速かつ正確に応答できるようになります。

---

### チャット UX の改善

チャットの UX に複数の改善が行われました。

- **コード差分のインライン表示**: コード差分がチャット会話内に直接レンダリングされるようになり、別の差分ビューに切り替えることなく提案された変更をレビューできます。
- **レンダリングパフォーマンスの向上**: チャット応答のレンダリングが高速化され、レイアウトスラッシングの削減やストリーミング中の効率的なインクリメンタル更新が行われました。ツール呼び出し更新の急速なバーストが拡張ホストで一時的なハングを引き起こす問題も修正されました。
- **チャット送信パフォーマンスの改善**: チャットカスタマイズの読み込みによってチャットメッセージの送信がブロックされる問題が修正されました。プロンプトがまだ読み込み中でも、メッセージが即座にチャット会話に表示されるようになりました。
- **サブエージェント進捗の視覚的改善**: サブエージェントの進捗の展開ビューがより視覚的に区別しやすくなり、サブエージェントの進行状況を追いやすくなりました。

---

### JS/TS Chat Features 拡張機能 (Preview)

**設定**: `jsts-chat-features.skills.enabled`

新しい組み込み拡張機能 **JS/TS Chat Features** が追加されました。Copilot のスキルを強化し、TypeScript および JavaScript との連携を向上させます。初回リリースでは、モダンな TypeScript プロジェクトのセットアップに関するスキルが提供されます。今後のリリースで機能の拡張が予定されています。

この機能を試すには、設定 `jsts-chat-features.skills.enabled` を有効にしてください。

---

### GitHub Pull Requests 拡張機能 — プルリクエスト作成のチャットツール

[GitHub Pull Requests](https://marketplace.visualstudio.com/items?itemName=GitHub.vscode-pull-request-github) 拡張機能に、プルリクエストを作成するためのチャットツールが追加されました。また、「Delete Local Branches and Remotes」コマンドから worktree の削除も可能になりました。

---

### Edit Mode の非推奨化スケジュール（継続）

VS Code v1.110 で正式に非推奨となった **Edit Mode** について、引き続きスケジュールが案内されています。設定 `chat.editMode.hidden` を使用して一時的に Edit Mode を再有効化することができますが、この設定は v1.125 までのサポートとなります。v1.125 以降、Edit Mode は完全に削除され、設定による有効化もできなくなります。
