# Visual Studio Code v1.117.0 (2026年4月) — GitHub Copilot 関連アップデート

## リリース日

2026-04-22

## VS Code バージョン

1.117.0

## 概要

VS Code v1.117.0 では、Copilot Business / Enterprise ユーザー向けに自前の API キーでモデルを接続できる BYOK 機能が追加されたほか、チャット応答のインクリメンタルレンダリング、エージェントセッション管理の改善、ターミナルでの Copilot CLI 起動の修正など、エージェント体験を中心に改善が行われています。

---

## GitHub Copilot 関連の変更点

### Copilot Business / Enterprise 向け BYOK（Bring Your Own Key）

チームがコンプライアンス、パフォーマンス、コストなどの理由で特定のモデルを必要とする場合に、ツールを切り替えることなく利用できるようになりました。BYOK 機能により、Copilot Business および Enterprise ユーザーは OpenRouter、Ollama、Google、OpenAI などのプロバイダーの API キーを接続し、VS Code チャットで直接使用できます。

デフォルトで BYOK は有効化されており、管理者は GitHub.com の [Copilot ポリシー設定](https://github.com/settings/copilot/features) にある **Bring Your Own Language Model Key** ポリシーで無効化できます。これにより、組織で利用可能なモデルプロバイダーを管理者が制御しつつ、開発者は既存のワークフロー内でモデルを利用できます。

ポリシーが有効化された後、組織メンバーは[組み込みプロバイダーからモデルを追加](https://code.visualstudio.com/docs/copilot/customization/language-models#_bring-your-own-language-model-key)するか、言語モデルプロバイダー拡張機能をインストールできます。

---

### チャット応答のインクリメンタルレンダリング (Experimental)

チャット応答がブロック単位でストリーミングされ、オプションのアニメーション付きでより流動的かつ自然に表示されるようになりました。従来のタイマーベースのレンダリングとは異なり、各ブロックが準備でき次第レンダリングされるため、長い応答の体感待ち時間が短縮されます。

以下の設定でインクリメンタルレンダリングを構成できます。

- `chat.experimental.incrementalRendering.enabled`: インクリメンタルレンダリングの有効/無効を切り替え。デフォルト: `true`
- `chat.experimental.incrementalRendering.animationStyle`: アニメーションスタイルの設定。オプション: `none`、`fade`、`rise`、`blur`、`scale`、`slide`、`reveal`。デフォルト: `fade`
- `chat.experimental.incrementalRendering.buffering`: レンダリング前のコンテンツバッファリング方法の設定。低いバッファリングレベルではレンダリングが速くなりますが、不完全な文や部分的な Markdown が表示される場合があります。オプション: `off`、`word`、`paragraph`。デフォルト: `off`

### エージェントセッションのソート機能

多数のエージェントセッションが蓄積された場合に、目的のセッションを見つけやすくなりました。**Agent Sessions** ビューで、セッションを作成日時または最終更新日時でソートできるようになり、中断した作業を素早く再開できます。

---

### バックグラウンドターミナルコマンドのシステム通知

エージェントがバックグラウンドで長時間実行されるターミナルコマンドを実行している場合、進行状況を把握しにくいことがありました。これらのコマンドがチャット応答内に **システム通知** として表示されるようになり、ターミナルに切り替えることなくステータスを監視できます。

---

### Visual Studio Code Agents アプリ (Insiders) — アップデート

VS Code Insiders に同梱される **Visual Studio Code Agents** アプリに、以下のアップデートが行われました。

- **サブセッションの作成**: セッションタイトルの **+** を選択すると、現在のセッションからサブセッションを作成できます。並行リサーチやコードレビューなど、コンテキストを維持したまま追加作業を開始するのに便利です。
- **インライン変更レンダリングの改善**: エージェントがコードを編集した際の差分表示が改善され、変更のスキャンと比較が容易になりました。
- **アップデート体験の改善**: OS 間でのアップデートフローが改善され、最新バージョンへのアップデートがよりスムーズになりました。
- **テーマ・チャット応答・UX の改善**: テーマ、セッションリスト、応答レンダリング、全体的な UX の継続的な改善。

### Copilot CLI のカスタムターミナルプロファイルでの起動

Copilot CLI ターミナルプロファイルが、デフォルトのターミナルプロファイルが `fish`（macOS / Linux）や Git Bash（Windows）などの非デフォルトシェルに設定されている場合でも、ターミナルパネルから起動できるようになりました。

以前は、この構成で **GitHub Copilot CLI** をターミナルプロファイルピッカーから選択すると、`No terminal profile options provided for id 'copilot-cli'` エラーが発生し、ターミナルが起動しませんでした。

---

### エージェント CLI のターミナルタイトル表示

Copilot CLI、Claude Code、Gemini CLI などのエージェント CLI は通常 `node` プロセスとして動作するため、ターミナルタイトルに汎用的な `node` ラベルが表示されていました。ターミナルがこれらのエージェント CLI を独立したシェルタイプとして検出し、CLI が出力する OSC タイトルシーケンスをターミナルタイトルとして使用するようになったため、各ターミナルがどのエージェントをホストしているか明確に識別できます。

この改善は macOS、Linux、Windows 上の Copilot CLI、Claude Code、Gemini CLI に対応しています。Codex は macOS では OSC タイトルシーケンスを出力しないため、現時点では検出されません。この動作はデフォルトで有効であり、設定 `terminal.integrated.tabs.allowAgentCliTitle` で切り替えることができます。
