# Visual Studio Code v1.118.0 (2026年4月) — GitHub Copilot 関連アップデート

## リリース日

2026-04-29

## VS Code バージョン

1.118.0（1.118.1 まで Recovery リリース済み）

## 概要

VS Code v1.118.0 では、Copilot CLI セッションのリモートコントロール、セマンティックインデックスの全ワークスペースへの展開、トークン効率の大幅改善など、エージェント体験の拡張と効率化が中心のリリースとなっています。

---

## GitHub Copilot 関連の変更点

### Visual Studio Code Agents アプリ (Insiders) — アップデート

VS Code Insiders に同梱される **Visual Studio Code Agents** アプリに、以下のアップデートが行われました。

- **VS Code とのステート共有**: Agents アプリが VS Code Insiders とより多くのステートを共有するようになり、認証（Windows）、AI カスタマイズ、ワークスペース信頼、最近のフォルダー、キーボードショートカットが共有されます。
- **Claude Agent**: Agents アプリで Claude Agent が利用可能になり、Copilot CLI や Copilot Cloud と並行して使用できます。
- **Web クライアント**: [insiders.vscode.dev/agents](https://insiders.vscode.dev/agents) からブラウザで Agents 体験にアクセスできます。Dev Tunnel を実行中の任意のマシンからエージェントネイティブなワークフローを利用できます。
- **バックグラウンドブラウザ**: 統合ブラウザがセッション間で保持されるようになり、セッションに戻った際にリフレッシュされなくなりました。
- **変更のレイアウトコントロール**: エージェントが変更を行った際、差分ビューを Chat ビューとサイドバイサイドで開くか、モーダルウィンドウで開くか選択できます。
- **動的タイトルバーエントリーポイント**: VS Code Insiders のタイトルバーから **Open in Agents** を選択して Agents アプリに移動し、逆に **Open in VS Code** で戻れるようになりました。

### Copilot CLI セッションのリモートコントロール (Experimental)

**設定**: `github.copilot.chat.cli.remote.enabled`

Copilot CLI セッションを、GitHub.com や GitHub モバイルアプリから遠隔で監視・操作できるようになりました。以前は、エージェントが承認や質問で一時停止した場合、マシンに戻るまで作業が停滞していましたが、リモートコントロールにより、別のデバイスから進捗の確認、承認への応答、作業の方向づけが可能になります。

使用方法:
1. `github.copilot.chat.cli.remote.enabled` 設定を有効にする
2. Copilot CLI チャットで `/remote on` と入力する

`/remote` でリモートコントロールの状態を確認、`/remote off` で無効化できます。

---

### Copilot CLI のセッションタイトル同期

Copilot CLI セッションのタイトルが、チャットセッションリスト、チャットエディタータブ・ヘッダー、ターミナルインターフェースなど、すべての表示面で一貫して同期されるようになりました。

VS Code は Copilot SDK のセッションタイトル API を信頼できるソースとして採用し、セッションリストとチャットエディターヘッダーを単一のタイトルリゾルバーを通じて処理します。どこでリネームを行っても、すべての表示面でタイトルが同期されます。ターミナルから Copilot CLI で行ったリネームも、次回セッションメタデータが読み込まれた際に VS Code に反映されます。

---

### Git AI 共同オーサリングのデフォルト有効化

VS Code はチャットおよびエージェントワークフローで Git AI 共同オーサリングをデフォルトで有効にしました。Copilot がファイルに変更を加えた場合、そのコミットに Copilot が共同著者として自動的に追加されます。

デフォルト動作は `git.addAICoAuthor` 設定で変更できます。

---

### セマンティックインデックスの全ワークスペースへの展開

セマンティックインデックスがすべてのワークスペースで利用可能になりました。以前は GitHub または ADO リポジトリを使用するワークスペースに限定されていましたが、ローカルリポジトリなど任意のワークスペースでも意味検索が利用できます。

セマンティックインデックスは自動的に構築・維持されます。GitHub や ADO リポジトリを使用するワークスペースでは即座にセマンティック検索が利用でき、それ以外のワークスペースでは初回インデックスの構築に数分かかる場合があります。**Build Codebase semantic index** コマンドで明示的にインデックスを構築することも可能です。

---

### GitHub テキスト検索（リポジトリ・組織横断）

エージェントが現在のワークスペース外のコードで正確な文字列、API 名、エラーメッセージを検索する必要がある場合に対応する、組み込み `githubTextSearch` エージェントツールが追加されました。GitHub リポジトリまたは GitHub 組織全体のコードに対して grep スタイルの検索を実行できます。

既存の `githubRepo` ツール（セマンティック検索）と組み合わせて使用することで、現在作業中のコードベース以外のコードからも学習できるようになります。

---

### スキルの専用コンテキスト (Experimental)

**設定**: `github.copilot.chat.skillTool.enabled`

マルチステップのツール呼び出しや大量の参照資料を取り込むスキルを使用する際、その補助コンテンツがメインチャットのコンテキストを圧迫し、フォローアップ応答の品質を低下させることがありました。

スキルを専用のサブエージェントコンテキストで実行し、メインの会話から実行を分離できるようになりました。これにより、プライマリコンテキストが集中した状態を維持し、スキル応答の品質が向上します。

`SKILL.md` のフロントマターで `context: fork` 属性を設定することで有効にできます。

---

### ワークスペース `.mcp.json` ファイルとサーバー重複排除

ワークスペースレベルの `.mcp.json` ファイルによる MCP サーバーの宣言がサポートされ、Copilot CLI などの他のツールと整合されました。

同じ名前を共有する MCP サーバーの重複排除動作も導入されました。デフォルトでは最も具体的な MCP サーバーのみが有効になり、サーバーを有効にすると同じ名前の他のサーバーが無効になります。

### トークン効率の改善

GitHub Copilot が 2026年6月1日から[使用量ベースの課金](https://github.blog/news-insights/company-news/github-copilot-is-moving-to-usage-based-billing/)に移行することに伴い、エージェントの品質を損なうことなくトークン効率を改善する複数の施策が実施されています。

#### プロンプトキャッシュ効率

システムプロンプト、ツール、会話履歴、要約全体でキャッシュの再利用が改善されました。繰り返されるコンテキストがはるかに低いトークンレートで課金されるため（Anthropic モデルの場合約10分の1）、長いマルチターンエージェントワークフローでのコスト削減に貢献します。

- **戦略的キャッシュブレークポイント配置**: システムプロンプト末尾、ツール末尾、最新ツールターン末尾、会話ターン境界にキャッシュブレークポイントが設置され、エージェントセッション中は各リクエストの93%以上がキャッシュから再利用されます。
- **キャッシュ安定なシステムプロンプトとツールリスト**: リクエスト間のバイトドリフトの原因を排除し、ツールリストの順序を一貫させることでキャッシュの無効化を防止します。`chat.experimental.symbolTools.cacheStable` 設定により、言語拡張の有効化によるキャッシュリセットを回避できます。
- **キャッシュフレンドリーなバックグラウンド圧縮**: セッションが長くなると古いターンをバックグラウンドで要約し、メインエージェントと同じキャッシュされたコンテキストを再利用します。
- **最後の2メッセージブレークポイント戦略**: `github.copilot.chat.anthropic.cacheBreakpoints.lastTwoMessages` 設定で利用可能です。

#### ツール検索ツール

エージェントのツールセットを2つのグループに分割し、リクエストを軽量に保ちます。約30の**常時利用可能な**コアツール（ツール呼び出しの約88%をカバー）が常に含まれ、残りの**遅延ツール**はモデルが明示的に要求するまでスキーマがロードされません。

Anthropic モデル（Claude Sonnet 4.5+ および Opus 4.5+）ではデフォルトで有効で、最大**20%のトークン削減**が確認されています。OpenAI モデル（GPT-5.4 および GPT-5.5）では `github.copilot.chat.responsesApi.toolSearchTool.enabled` 設定で有効にできます。

#### エージェント検索ツール

コードベース探索とコンテキスト取得を処理する専用ツールです。メインエージェントがコンテキストを必要とする場合、自然言語で検索内容を記述すると、小型のファインチューニング済みモデルが grep、ファイル検索、セマンティック検索、ファイル読み取りを並列実行し、最も関連性の高い結果を返します。最大20%のトークン削減が期待されます。

#### エージェント実行ツール

ターミナルコマンドの実行を処理する専用ツールです。テストの実行やビルドの確認が必要な場合、このツールにタスクを委任します。1回の呼び出しにつき最大10回のターミナル呼び出しに制限されています。冗長な出力をフィルタリングし、コーディングエージェントに必要な部分のみを返すことで、トークン使用量を削減します。

### OpenAI モデルの WebSocket サポート

対応する OpenAI モデルでは、チャットリクエストに Responses API の [WebSocket モード](https://developers.openai.com/api/docs/guides/websocket-mode)が使用されるようになりました。ターンごとに新しい HTTP リクエストを開く代わりに、永続的な WebSocket 接続を維持し、新しい入力アイテムと前回のレスポンス ID のみを送信します。

これにより、特にエージェントワークフローでのリクエストサイズとレイテンシが削減され、OpenAI モデルが**12%高速化**されました。対応モデルで自動的に使用され、設定は不要です。

### Chronicle (Experimental)

**設定**: `github.copilot.chat.localIndex.enabled`

チャット履歴をローカル SQLite データベースに記録し、コーディング活動の検索や要約を可能にする機能です。セッションメタデータ（ブランチ、リポジトリ、タイムスタンプ）、会話ターン、ツール呼び出しで操作したファイル、外部参照（PR、Issue、コミット）を記録します。

利用可能なコマンド:
- `/chronicle:standup`: 過去24時間のコーディングセッションからスタンドアップレポートを生成
- `/chronicle:tips`: 7日間の使用状況を分析し、プロンプティング、ツール使用、ワークフローに関するパーソナライズされたヒントを提供
- `/chronicle [query]`: セッション履歴に対する自然言語クエリ（例: 「昨日編集したファイルは？」）

### 承認済みアカウント組織ポリシー

エンタープライズは `ChatApprovedAccountOrganizations` デバイスポリシーを使用して、承認された GitHub 組織メンバーシップに基づいてチャットおよび関連 AI 機能のアクティベーションをゲートできるようになりました。

チャット機能は、(1) ユーザーが承認された組織にメンバーシップを持つ GitHub アカウントにサインインし、(2) アカウントベースのポリシーが解決されるまでアクティベートされません。このフェイルクローズド動作は、GitHub.com でアカウントベースのポリシーを設定しているエンタープライズに有用です。

---

### サンドボックスのデフォルト読み取り権限の強化

`$HOME` ディレクトリ下のすべてのパスに対する読み取りアクセスが自動的に有効になることがなくなりました。コマンド実行前に、実行コマンドに基づいてのみ読み取り権限が追加され、`$HOME` ディレクトリ内の他のすべてのパスは読み取りアクセスが拒否されます。

デフォルトでは、ワークスペースフォルダーとサンドボックス一時フォルダーのみが `$HOME` 下で読み取りアクセスを許可されます。

---

### Chat Customizations Evaluation 拡張機能

プロンプトファイル、カスタムエージェント、インストラクション、スキルなどのチャットカスタマイズを分析・改善するための新しい拡張機能 **Chat Customizations Evaluations**（拡張機能 ID: `ms-vscode.vscode-chat-customizations-evaluations`）が追加されました。

カスタマイズファイルを分析し、検出された問題に対する診断情報と改善推奨を生成します。プロンプト、エージェント、インストラクション、スキル定義ファイルを開いて **Analyze** を選択すると評価が実行されます。

---

### ターミナルからの質問カルーセルのキーボードショートカット

ターミナル操作をきっかけに Copilot が質問カルーセルを表示した際、⌥T（Windows/Linux: Alt+T）でターミナルにフォーカスを戻せるようになりました。以前は **Focus Terminal** ボタンを選択するしか方法がありませんでした。

`accessibility.verbosity.chatQuestionCarousel` 設定でナビゲーションヒントの表示を制御できます。
