# Visual Studio Code v1.119.0 (2026年5月) — GitHub Copilot 関連アップデート

## リリース日

2026-05-06

## VS Code バージョン

1.119.0

## 概要

VS Code v1.119.0 では、エージェントとブラウザタブの共有機能の強化、OpenTelemetry によるエージェントセッションのトレーシング、トークン使用量の最適化など、エージェント体験の可観測性と効率性の向上が中心のリリースとなっています。

---

## GitHub Copilot 関連の変更点

### エージェントとのブラウザタブ共有

エージェントがライブブラウザにアクセスできると、リアルタイムで変更を検証し、より速く反復できるようになります。Web 開発では、エージェントがコードを編集し、ページをリロードして、1ターンで修正を確認できます。

エージェントは[統合ブラウザ](https://code.visualstudio.com/docs/debugtest/integrated-browser)に自動的にアクセスすることはできず、明示的にブラウザページを共有する必要があります。

今回のリリースでは、以下の新しいブラウザ共有方法が追加されました：

- **ブラウザタブをコンテキストとしてアタッチ**: サジェストコンテキスト、コンテキストピッカー、ドラッグ＆ドロップを通じて、ブラウザタブをチャットに明示的にアタッチできるようになりました。アタッチされたブラウザタブは共有状態に入り、エージェントがページを読み取り・操作できます。
- **エージェントからのタブ共有リクエスト**: エージェントは開いているブラウザタブの数と未共有のタブの情報を取得できるようになり、ページを操作する必要がある場合にタブの共有をリクエストできます。ユーザーはプロンプトで承認または拒否できます。
- **既存タブの再利用プロンプト**: エージェントが既存の未共有タブと同じドメインで新しいタブを開こうとした場合、既存タブの再利用を提案するプロンプトが表示されます。

### Visual Studio Code Agents アプリ (Insiders) — アップデート

VS Code Insiders に同梱される **Visual Studio Code Agents** アプリに、以下のアップデートが行われました。

- **新セッションリポジトリピッカーの再設計**: 新しいセッション開始時に、ローカルフォルダー、リポジトリ、リモートオプション間を簡単に切り替えられるようになりました。
- **サブセッションの改善**: サブセッションの作成と管理が改善され、サブセッションタブやライフサイクル処理の修正が行われました。
- **Web・モバイルの改善**: [Agents Web クライアント](https://insiders.vscode.dev/agents)のブラウザ体験がデスクトップ体験により近づくよう改善が続けられています。モバイル体験も改善され、モバイルデバイスのブラウザからセッションの作成・管理および変更の確認が可能になりました。
- **環境管理と継続性**: VS Code と Agents 間の接続と環境管理の進化に継続的に投資されています。
- **進捗 UX**: エージェントがタスクに取り組んでいる際の進捗がより視覚的になり、回転する進捗メッセージとチャット入力ボックスのボーダーアニメーションが追加されました。
- **開発者の楽しみ**: 新しいセッションページにイースターエッグが追加されました。`sessions.developerJoy.enabled` を有効にすると確認できます。

### エージェントセッションの OpenTelemetry トレーシング

**設定**: `github.copilot.chat.otel.enabled`、`github.copilot.chat.otel.otlpEndpoint`

エージェントセッションが長く自律的になるにつれ、エージェントが何をしたか、各ステップにかかった時間、トークンの消費量を理解することが、コストの最適化や予期しない動作のデバッグに不可欠になります。

Copilot Chat エージェントセッション（ローカルエージェント、Copilot CLI バックグラウンドエージェント、Claude エージェントを含む）が、[GenAI セマンティック規約](https://github.com/open-telemetry/semantic-conventions/blob/main/docs/gen-ai/)に準拠した OpenTelemetry トレース、メトリクス、イベントを出力するようになりました。OTLP 互換バックエンド（例：[Aspire Dashboard](https://aspire.dev/dashboard/standalone/)）でエージェントの動作、レイテンシ、トークン使用量を監視できます。

各ユーザーリクエストは `invoke_agent` ルートスパン（例：`invoke_agent claude`）を生成し、ネストされた `chat`、`execute_tool`、`execute_hook` 子スパンを持ちます。サブエージェントの呼び出しは、呼び出し元エージェントの `execute_tool` スパンに自動的に親子関係が設定され、単一の接続されたトレースでエージェントの作業全体を可視化できます。

### Copilot CLI および Claude エージェントのレスポンスにモデル詳細を表示

**設定**: `github.copilot.chat.agent.modelDetails.enabled`

[Copilot CLI](https://code.visualstudio.com/docs/copilot/agents/copilot-cli) および [Claude エージェント](https://code.visualstudio.com/docs/copilot/agents/third-party-agents) のレスポンスに、使用されたモデルとその乗数がバッジとして表示されるようになりました。バッジは各レスポンスの完了時にライブで表示され、ウィンドウのリロードは不要です。セッション途中でモデルを切り替えた場合も更新されます。

Copilot CLI で **Auto** モデル選択を使用している場合、バッジには `auto` ではなく実際に使用されたモデル（例：`Claude Sonnet 4.6`）が表示されます。解決されたモデル情報はセッション履歴から再構築された場合も保持されます。

この機能はデフォルトで有効です。バッジを非表示にするには `github.copilot.chat.agent.modelDetails.enabled` 設定を無効にしてウィンドウをリロードしてください。

### Todo リスト管理のトークン使用量最適化（実験的機能）

**設定**: `github.copilot.chat.agent.backgroundTodoAgent.enabled`

Todo リストはエージェントが複雑なマルチステップタスクを追跡するのに役立ちますが、メインモデルが Todo リストを更新するたびにトークンを消費し、長いセッションではコストが増加します。

Todo リスト管理を軽量なバックグラウンドエージェントに委任することで、メインモデルは実際のタスクに集中し、小さなモデルが進捗追跡を同期させます。これにより、エージェントの集中力を維持しつつ、全体的なトークン使用量が削減されます。

この設定を有効にすると、バックグラウンドエージェントがメインエージェントのアクティビティを監視し、完了中・進行中の作業を反映するよう Todo リストを更新します。メインエージェントは Todo ツールを持たなくなるため、会話のトークンコストが節約されます。

> **注意**: Todo ツールが手動でチャットリクエストに追加された場合（例：`#todo`）、またはカスタムエージェントがツールリストに指定している場合、バックグラウンドエージェントはオーバーライドされ実行されません。

この機能はデフォルトで無効です。`github.copilot.chat.agent.backgroundTodoAgent.enabled` 設定で有効にできます。

### 使用量ベースの課金アップデート

GitHub Copilot は 2026年6月1日から[使用量ベースの課金](https://docs.github.com/en/copilot/concepts/billing/usage-based-billing-for-individuals)に移行します。今回のリリースでは、チャットステータスダッシュボード、チャット入力通知、モデルピッカーに課金・クレジット情報の表示をサポートする内部変更が含まれています。これらの UI 更新はまだユーザーには表示されず、使用量ベースの課金の展開時に有効になります。

### エージェントサンドボックスのネットワークアクセス許可

**設定**: `chat.agent.sandbox.enabled`

エージェントサンドボックスはツールのアクセスを制限してシステムを保護しますが、厳格なネットワークブロックはパッケージのインストール、API 呼び出し、開発サーバーの実行時に支障をきたすことがあります。

`chat.agent.sandbox.enabled` 設定に `allowNetwork` モードが追加されました。ファイルシステムの制限を維持しつつネットワークドメインのブロックを解除するため、サンドボックスの保護を維持しながらネットワークアクセスの中断を減らすことができます。

```json
"chat.agent.sandbox.enabled": "allowNetwork"
```

ネットワークアクセスが許可されている場合、`chat.agent.allowedNetworkDomains` および `chat.agent.deniedNetworkDomains` 設定は無視されます。

### セッション許可コマンドの一時フォルダへの書き込み自動承認

**設定**: `chat.tools.terminal.blockDetectedFileWrites`

`chat.tools.terminal.blockDetectedFileWrites` がデフォルト値の `outsideWorkspace` に設定されている場合、ワークスペース外にファイルを書き込むターミナルコマンドは **Allow All Commands in Session** を選択していても承認が必要でした。

今回のリリースでは、**Allow All Commands in Session** がアクティブな場合、OS の一時フォルダ（macOS/Linux では `/tmp`、Windows では `%TEMP%`）への書き込みはこのチェックから免除されるようになりました。

これにより、一時フォルダにスクラッチファイルを保存する一般的なエージェントワークフローでセッションが中断されなくなり、ワークスペース外の他の場所への書き込みは引き続き確認が必要です。

---

### Edit Mode の非推奨化（継続）

**Edit Mode** は VS Code バージョン 1.110 で正式に非推奨となりました。ユーザーは `chat.editMode.hidden` 設定を通じて一時的に Edit Mode を再有効化できます。この設定はバージョン 1.125 までサポートされ、バージョン 1.125 以降は Edit Mode が完全に削除され、設定による有効化もできなくなります。
