# Visual Studio Code v1.120.0 (2026年5月) — GitHub Copilot 関連アップデート

## リリース日

2026-05-13

## VS Code バージョン

1.120.0

## 概要

VS Code v1.120.0 では、Agents ウィンドウが Stable に昇格し、BYOK モデルのトークン使用量表示や思考労力設定、ターミナルコマンドのリスク評価やターミナル出力の圧縮など、エージェント体験の安全性と効率性が大幅に向上しています。

---

## GitHub Copilot 関連の変更点

### Agents ウィンドウが Stable に昇格（プレビュー）

VS Code のエージェント駆動開発のための新しいウィンドウタイプ「**Agents**」が、Insiders から Stable に昇格しました。Agents ウィンドウは、複数のプロジェクトにまたがるエージェントタスクの探索・反復・レビューを行うための専用スペースを提供します。

タイトルバーの「Open in Agents」ボタンなどから開くことができます。

今回のリリースでは以下の改善が行われました：

- **設定の保持**: ドロップダウン（エージェントハーネス、分離モードなど）での選択が新しいセッション作成時にも保持されるようになりました。
- **変更の破棄が容易に**: Changes パネルから直接編集を破棄できるようになりました。
- **上流の変更を同期**: Files パネルの同期ボタンでベースブランチからの上流変更を確認し、取り込めるようになりました。
- **変更操作の決定性向上**: Changes パネルのアクションが決定的になり、より高速に完了するようになりました。
- **完了セッションの全変更表示**: 完了済みセッションを開くと、エージェントの全編集内容が一目で確認できるようになりました。
- **最近のセッション間のナビゲーション**: タイトルバー左上の矢印ボタンで、ウィンドウを離れずに最近のセッション間を移動できるようになりました。
- **ウィンドウごとの設定オーバーライド**: Agents ウィンドウは VS Code の全設定を共有しつつ、特定の設定を Agents ウィンドウ専用にオーバーライドできるようになりました。

#### 拡張機能のサポート

テーマ、グラマー、言語、キーバインディングのみを提供する静的な拡張機能は、Agents ウィンドウで自動的にアクティベートされます。その他の拡張機能は `extensions.supportAgentsWindow` 設定で ID を指定してオプトインできます。

```json
"extensions.supportAgentsWindow": {
    "myextension.id": true
}
```

### Copilot CLI プラグインの自動検出

[エージェントプラグイン](https://code.visualstudio.com/docs/copilot/customization/agent-plugins)が GitHub Copilot CLI でインストールされた場合、VS Code が自動的に検出するようになりました。`copilot plugin install` を一度実行するだけで、CLI と VS Code の両方で利用可能になります。

以前は、同じプラグインを VS Code で別途インストールするか、`chat.plugins.paths` にパスを追加する必要がありました。

---

### BYOK モデルのトークン使用量表示

BYOK（Bring Your Own Key）で追加したモデルのトークン使用量が、Chat ビューのコンテキストウィンドウコントロールに正確に表示されるようになりました。

以前は、BYOK モデルを使用した場合、コントロールは常に 0% とゼロトークンを表示していました。今回のリリースにより、BYOK モデルでも正確なトークン使用量とコンテキストウィンドウの使用率が表示されます。

これにより、モデルのコンテキストウィンドウの管理が容易になり、コスト管理やレスポンス品質の最適化に役立ちます。

---

### BYOK 推論モデルの思考労力設定

BYOK の推論対応モデルに対して、Chat ビューのモデルピッカーから直接「思考労力（thinking effort）」を設定できるようになりました。この設定により、レスポンスの品質とレイテンシ・コストのトレードオフを調整できます。

選択した思考労力はすべてのリクエストでモデルに転送されます。

> 対象: OpenAI 互換エンドポイント（OpenAI、xAI (Grok)、OpenRouter、カスタム OpenAI / Azure OpenAI デプロイメント）経由で提供される BYOK 推論モデル。Anthropic モデルは以前からサポート済みで、今回プロバイダー間で一貫したコントロールが提供されるようになりました。

---

### モデルピッカーのプロバイダー別グループ化

Chat ビューのモデルピッカーが、プロバイダーごとにモデルをグループ化して表示するようになりました。複数のソースからモデルにアクセスしている場合に、目的のモデルを見つけやすくなります。名前での検索も可能です。

最近使用したモデルには、モデル名の横にプロバイダー名がグレーテキストで表示されるため、異なるプロバイダーの同名モデルを素早く区別できます。

> **ヒント**: チャット入力で `/models` と入力すると、モデルに素早くアクセスできます。

### ターミナルツール出力の圧縮（プレビュー）

**設定**: `chat.tools.compressOutput.enabled`

`git diff`、`ls -l`、`npm install` などのコマンドの長いターミナル出力が、モデルのコンテキストウィンドウの大部分を消費し、コードやエージェントの推論に使える余地が減少する問題に対処します。

`chat.tools.compressOutput.enabled` 設定を有効にすると、VS Code がコマンド出力をモデルに送信する前に後処理を行います：

- diff 内の変更のない大きなハンクを折りたたむ
- lockfile やスナップショットの diff を除去
- `ls -l` をエントリ名のみに縮小
- `npm install` のプログレスバー、非推奨警告、監査サマリーを除去

圧縮された出力には短いバナーが付与され、モデルはどのフィルターが適用されたかを確認し、必要に応じて圧縮を無効にできます。

### ターミナルコマンドのリスク評価（実験的機能）

**設定**: `chat.tools.riskAssessment.enabled`

ターミナルコマンドの確認プロンプトに、AI が生成したリスクバッジと、コマンドの動作の説明が表示されるようになりました。コマンドを詳しく確認すべきかを素早く判断できます。

各バッジは 3 つのレベルのいずれかと、コマンドに合わせた 1 文の要約を表示します：

- **Safe**（緑）: ファイルの読み取りや出力の表示のみで、変更を加えません。
- **Caution**（オレンジ）: ワークスペースの変更、パッケージのインストール、またはネットワーク経由のデータ送信を行います。
- **Review carefully**（赤）: 元に戻すのが困難または不可能なアクション（リモートへの強制プッシュやワークスペース外のファイル削除など）を実行します。

---

### Claude および Copilot CLI のプランモード制御

**設定**: `chat.planWidget.inlineEditor.enabled`

Claude エージェントまたは Copilot CLI でプランモードを使用する際に、VS Code がインラインプラン制御を表示し、エージェントが実行を開始する前にプランをレビュー・調整できます。今回のリリースでは以下の改善が行われました：

- **プランのインライン編集**: プランの編集が、別のエディタータブではなくコントロール内のインラインエディターで行えるようになり、コンテキストを失わずに反復できるようになりました。
- **フィードバックモードの明確化**: プランにフィードバックを提供する際、フィードバックモードであることがより明確に表示され、追加済みのフィードバックも表示されるようになりました。
- **インラインエディターの無効化**: `chat.planWidget.inlineEditor.enabled` 設定でインライン編集を無効にし、通常のエディタータブでの編集にフォールバックすることも可能です。
