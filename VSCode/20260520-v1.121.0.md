# Visual Studio Code v1.121.0 (May 2026) — GitHub Copilot 関連アップデート

## リリース日

2026-05-20

## VS Code バージョン

1.121.0

## 概要

リモートマシン上でのエージェントセッション実行、ユーティリティモデルの設定、ターミナルツールの最適化など、GitHub Copilot のエージェント機能を大幅に強化するリリース。

---

## GitHub Copilot 関連の変更点

### Agents ウィンドウの改善（プレビュー）

前回リリースで VS Code Stable にプレビューとして導入されたエージェント駆動のコンパニオンウィンドウ「Agents ウィンドウ」の改善が継続されています。VS Code タイトルバーの **Open in Agents** ボタンなどから開くことができます。

拡張機能のストーリーについても引き続き取り組みが進められており、Agents ウィンドウでの拡張機能の有効化やその動作について、GitHub Issues を通じたフィードバックが求められています。

---

### リモートエージェント（プレビュー）

Agents ウィンドウで、自身が所有するリモートマシン上でエージェントセッションを実行する実験的サポートが追加されました。

#### 接続方法

- **SSH**: 既存の `~/.ssh/config` エントリから選択するか、`user@host` を入力
- **Dev Tunnels**: ターゲットマシンで `code tunnel` を実行して作成済みのトンネルから選択

#### 動作の仕組み

Agents ウィンドウがリモートに接続し、VS Code CLI をダウンロード・インストール（SSH の場合）するか、Dev Tunnel 経由で実行中の CLI サーバーに接続します。「エージェントホスト」と呼ばれる軽量プロセスを起動し、[Copilot SDK](https://www.npmjs.com/package/@github/copilot-sdk) 上に構築された新しいエージェントループをホストします。

重要な点として、リモートエージェントホストは長寿命プロセスです。クライアントが切断されてもリモートでセッションが実行し続けるため、ラップトップを閉じてもリモートエージェントは作業を継続します。

#### Agent Host Protocol（AHP）

Agents ウィンドウとエージェントホスト間の接続は、**[Agent Host Protocol (AHP)](https://microsoft.github.io/agent-host-protocol/)** と呼ばれる新しいオープンプロトコルです。スタンドアロン仕様としてオープンに開発されています。

AHP の主要な設計原則は、複数のクライアント間でエージェントセッションの協調を可能にすることです。エージェントホストが権威的な状態を管理し、接続されたすべてのクライアントに同期し、純粋なリデューサーを通じてすべての変更を順序付けます。

---

### エージェントの可観測性：OpenTelemetry と Grafana

Azure Managed Grafana チームとの連携により、VS Code のエージェントが出力する OpenTelemetry シグナル用の構築済み Azure Managed Grafana ダッシュボードが提供されました。VS Code を OTel Collector に向け、Azure Application Insights に転送し、ダッシュボードをインポートすることで、エージェント操作、トークン使用量、チャットセッション、ツール呼び出し、モデルごとの応答時間と最初のトークンまでの時間（TTFT）を可視化できます。

---

### Claude エージェントの Auto パーミッションモード（プレビュー）

**設定**: `github.copilot.chat.claudeAgent.allowAutoPermissions`

Claude Agent が [Auto モード](https://code.claude.com/docs/en/permission-modes#eliminate-prompts-with-auto-mode) をサポートしました。これにより、Claude が許可プロンプトなしで実行できるようになります。別の分類リクエストがアクション実行前にレビューし、リクエストを超えるエスカレーション、認識されていないインフラストラクチャの対象化、または Claude が読み取った悪意のあるコンテンツによる駆動をブロックします。

長時間実行タスクでプロンプト疲労を軽減しつつ、バックグラウンドでの安全性チェックを維持したい場合に便利です。

> **注意**: 安全性チェックなしの完全な無人実行（「YOLO モード」）を希望する場合は、`github.copilot.chat.claudeAgent.allowDangerouslySkipPermissions` を有効にして「Bypass all permissions」を表示させることができます。

---

### ユーティリティモデルの設定

**設定**: `chat.utilityModel`、`chat.utilitySmallModel`

VS Code はバックグラウンドでユーティリティモデルを使用して、タイトル生成、要約、コミットメッセージ、リネーム提案、プロンプト分類、インテント検出などのチャット関連タスクを処理しています。デフォルトでは GitHub Copilot 提供のユーティリティモデルが使用されます。

これらのフローに対して、Bring Your Own Key (BYOK) モデルを含む利用可能なモデルを設定できるようになりました：

- `chat.utilityModel`: 一般的なユーティリティフローで使用するモデルをオーバーライド
- `chat.utilitySmallModel`: 高速で軽量なユーティリティフローで使用するモデルをオーバーライド（高速かつ低コストのモデルを推奨）

---

### Custom Endpoint プロバイダー（BYOK）（Insiders）

新しい BYOK プロバイダー「Custom Endpoint プロバイダー」が追加されました。Chat Completions、Responses、または Messages 互換の任意のエンドポイントを単一の設定から Copilot Chat に接続できます。Chat Completions のみをサポートしていたレガシーの OpenAI Compatible（`customoai`）プロバイダーを置き換えるもので、旧プロバイダーは廃止予定としてマークされています。

モデル追加時に、API ファミリー（`chat-completions`、`responses`、`messages`）を選択できます。

> **注意**: Custom Endpoint プロバイダーは現在プレビューで、VS Code Insiders でのみ利用可能です。

---

### エージェント対応のターミナルコマンド

コマンドラインツールがターミナルコマンドを人間が起動したのか VS Code のエージェントフローが起動したのかを判別できるようになりました。

VS Code はエージェントが起動したターミナルコマンドに `VSCODE_AGENT` 環境変数を設定するようになりました。CLI はこの変数をチェックして、機械可読出力への切り替え、プログレスアニメーションの抑制、セッションをブロックするプロンプトのスキップが可能です。

---

### ターミナルツールのバックグラウンド実行インジケーター

チャットターミナルコマンドがツール呼び出しの返却後も実行を継続している場合、チャット UI に **Running `<command>` in background - Show** と表示されるようになりました。**Show** アクションで基になるターミナルを表示・フォーカスできます。コマンド終了後、ヘッダーは通常の完了状態に戻ります。

これにより、特に非同期実行やタイムアウト後にバックグラウンド実行に昇格したコマンドについて、まだ実行中であることが明確になります。

---

### バックグラウンドエージェントターミナルのクリーンアップ

長時間のチャットセッションで複数のターミナルコマンドを実行すると、各コマンド完了後にバックグラウンドターミナルが蓄積し、ターミナルリストが古いエントリで埋まりリソースを消費する問題がありました。

VS Code は、チャットエージェントが作成したバックグラウンドターミナルをコマンド完了時に自動的に破棄するようになりました。コマンド出力はチャット UI に保持されます。**Show** でバックグラウンドターミナルを表示した場合は、引き続き検査やインタラクションのために開いたままになります。

---

### ターミナルツール出力の圧縮拡大

**設定**: `chat.tools.compressOutput.enabled`

`pytest`、`jest`、`cargo test`、`tsc`、パッケージインストールワークフローなどのコマンドは、重要な結果を表示する前に大量のプログレス出力を生成し、トークンを浪費してモデルが関連情報を見つけにくくしていました。

チャットターミナルツールは、より多くの種類の冗長なコマンド出力をモデルに送信する前に圧縮するようになりました。一般的なテストランナー、ビルドツール、リンター、Docker コマンド、パッケージマネージャーをカバーし、反復的なプログレス情報やその他の低価値な出力がより頻繁にトリムされます。

---

### 機密ターミナルプロンプトの保護

ターミナルコマンドでパスワード、パスフレーズ、PIN、または確認コードのプロンプトが表示された場合、エージェントがこれらの秘密情報を誤ってキャプチャまたはリプレイするリスクがありました。

チャットターミナルコマンドが機密プロンプトに到達すると、VS Code がそれをインターセプトします。デフォルトの権限モードでは、ターミナルにフォーカスして直接秘密情報を入力できる確認ダイアログが表示されます。自動承認フローでは、VS Code がコマンドをキャンセルし、モデルにリトライや秘密情報の要求をしないよう指示します。

これにより、資格情報がチャットコンテキストから排除され、エージェントが機密入力を誤って公開またはリプレイすることを防ぎます。

---

### クイック候補のデフォルト設定変更

Copilot のインライン提案は常にサジェストコントロールの選択に合わせて動作します。しかし、入力を始めると多くの場合、サジェストコントロールが入力文字で始まる最初の（アルファベット順の）グローバルシンボルを選択してポップアップしていました。これは実際に入力するテキストであることは稀で、Copilot にも不正確なプレフィックスに基づく提案をさせていました。

クイック候補のデフォルト設定（`editor.quickSuggestions`）が変更されました。インライン補完プロバイダー（Copilot など）が利用可能な場合、エディタで文字を入力してもサジェストコントロールが自動的にトリガーされなくなりました。その他の場合は従来通りポップアップします。

以前の動作に戻すには以下の設定を使用します：

```json
"editor.quickSuggestions": {
  "other": "on",
  "comments": "off",
  "strings": "off"
}
```

---

### 統合ブラウザでのチャットへの要素追加体験の改善

要素選択 UI が刷新され、より豊富な機能とテーマサポートが可能になりました。要素の範囲をクリック＆ドラッグで選択できるようになり、共有コンテナ要素のターゲットが容易になりました。また、ページ内の任意の場所を右クリックして要素をチャットに添付できるようになりました。
