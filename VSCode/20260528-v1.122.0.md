# Visual Studio Code v1.122.0 (May 2026) — GitHub Copilot 関連アップデート

## リリース日

2026-05-28

## VS Code バージョン

1.122.0

## 概要

エージェント体験のさらなる強化と BYOK（Bring Your Own Key）の柔軟性向上が中心のリリース。GitHub サインインなしでの BYOK 利用が可能になり、エアギャップ環境やオフライン環境でもチャット・ツール・MCP サーバーが利用可能に。

---

## GitHub Copilot 関連の変更点

### Agents Window（プレビュー）の改善

[Agents window](https://aka.ms/VSCode/Agents/docs) は、プロジェクト、ハーネス、マシンをまたいでエージェントセッションを探索・反復・レビューするための専用コンパニオンウィンドウ。今回のリリースでの改善点：

- **セッションホバー詳細**: セッションリスト内のセッションにカーソルを合わせると、詳細が一目で確認可能。使用ハーネスを示すアイコン付きのセッションタイトル、プロジェクト、ワークツリー、変更ファイルが表示される。
- **ローカル VS Code ハーネス（Insiders のみ）**: Agents window でローカルハーネスを使用する機能の改善を継続中。カスタムエージェントピッカーの改善を含む。`sessions.chat.localAgent.enabled` 設定で有効化可能。

---

### エージェント向け OpenTelemetry シグナルの強化

ローカルエージェントセッションが、[GitHub Copilot CLI の OpenTelemetry 規約](https://docs.github.com/en/copilot/reference/copilot-cli-reference/cli-command-reference#opentelemetry-monitoring)に準拠した `github.copilot.*` 属性名前空間を OpenTelemetry に出力するようになった。新しいシグナルにより、リポジトリコンテキスト、エージェントタイプ、構造化されたツールパラメータ、フック結果が各セッションに追加される。

詳細な属性リファレンスは [Monitor agent usage with OpenTelemetry](https://code.visualstudio.com/docs/copilot/guides/monitoring-agents) を参照。

---

### サンドボックスの動作変更

**設定**: `chat.agent.sandbox.enabled`（組織レベルで管理）

以前は **Bypass Approvals** または **Autopilot** モードでコマンドを実行した場合、まずサンドボックス内で実行を試み、非ゼロの終了コードで失敗した場合はサンドボックス外で自動的に再試行していた。承認がバイパスされている場合、これは実質的なセキュリティ上の利点がなく、動作の理解を困難にしていた。

Insiders ユーザーからのフィードバックに基づき、ターミナルサンドボックスは **Default Approvals** 使用時のみ適用されるよう変更された。安全性と有用性のバランスがより明確になった。

---

### GitHub サインインなしでの BYOK 利用

以前は VS Code で独自の言語モデル API キーを使用するには GitHub へのサインインが必要だった。今回のリリースで、[Bring Your Own Key（BYOK）](https://code.visualstudio.com/docs/copilot/customization/language-models#_bring-your-own-language-model-key) がサインインなしで動作するようになり、GitHub サインインが不可能なエアギャップ環境や制限された環境でもチャット、ツール、MCP サーバーが利用可能に。Ollama などのローカルモデルを使った完全オフラインワークフローも実現可能。

利用方法：コマンドパレットから **Manage Language Models** を実行し、Anthropic、Azure、Gemini、OpenAI、Ollama、OpenRouter、または[カスタムエンドポイント](https://code.visualstudio.com/docs/copilot/customization/language-models#_add-a-custom-endpoint-model)を追加。BYOK モデルが1つ以上構成されると、チャットビューが利用可能になり、サインインプロンプトは非表示になる。

> **注意**: インライン提案と Next Edit Suggestions（NES）は引き続き GitHub サインインが必要。BYOK はチャット、ツール、MCP サーバーのみに対応。

---

### ユーティリティモデル通知

**設定**: `chat.utilityModel`、`chat.utilitySmallModel`

チャットタイトル生成、コミットメッセージ生成、フィードバックなどの一部フローでは、通常 Copilot サブスクリプションから提供される小規模な[ユーティリティモデル](https://code.visualstudio.com/docs/copilot/customization/language-models#_change-the-model-for-utility-tasks)を使用している。サインアウト状態で BYOK を使用すると、デフォルトのユーティリティモデルに到達できないため、チャット入力に BYOK モデルを設定するよう促す通知が表示される。

2つのオプション：
- **Configure** を選択：設定を開き、`chat.utilityModel` と `chat.utilitySmallModel` に BYOK モデルを指定。独自の言語モデルを使用して全 AI 機能を利用可能に。
- 通知を閉じる：チャットのみ使用する場合。ユーティリティ駆動機能はモデルが構成されるまで非アクティブ。

両方のユーティリティ設定を構成するか、GitHub にサインインするか、すべての BYOK モデルを削除すると通知は自動的に非表示になる。

---

### カスタムエンドポイントプロバイダーが Stable で利用可能に

Chat Completions、Responses、または Messages API を実装するモデルに接続できるカスタムエンドポイントプロバイダーが VS Code Stable で利用可能に。セルフホスト、エンタープライズ、またはその他の互換性のある AI エンドポイントに接続してチャットを使用可能。

設定方法は [Add a custom endpoint model](https://code.visualstudio.com/docs/copilot/customization/language-models#_add-a-custom-endpoint-model) を参照。

---

### Agents window でのモデル管理

Agents window から直接 **Chat: Manage Language Models** コマンドを実行して、使用する言語モデルを設定可能に。ローカルセッションで動作し、同じフローで BYOK モデルも使用可能。モデル設定はエディタウィンドウと共有され、どちらで行った変更も両方に反映される。

---

### BYOK プロバイダーグループのきめ細かなアクション

Language Model エディタで、サポートされるプロバイダーグループがプロバイダースキーマに基づいたターゲットアクションを公開するようになった：**Update API Key**、**Add Model**、**Rename Group**、**Delete**。JSON 設定ファイルを手動で編集することなく、API キーのローテーションやプロバイダーグループの名前変更などの一般的なメンテナンスタスクをより迅速に実行可能。

---

### ブラウザスクリーンショットをチャットコンテキストとして追加

新しい **Add Screenshot to Chat** 機能により、統合ブラウザの現在のビューポートのスクリーンショットをチャットにコンテキストとして添付可能に。レイアウトの問題をデバッグするなど、UI 関連のタスクに特に便利。
