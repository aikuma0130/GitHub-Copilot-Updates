# Visual Studio Code v1.125.0 (June 2026) — GitHub Copilot 関連アップデート

## リリース日

2026-06-17

## VS Code バージョン

1.125.0

## 概要

Copilot の追加利用量ダッシュボード表示、言語モデルプロバイダーの拡張インストール、統合ブラウザの検索・リモート対応強化、エンタープライズ向け MDM 配信など、エージェント体験と管理機能を拡充するリリース。

---

## GitHub Copilot 関連の変更点

### 追加利用量（Additional Spend）の使用状況を VS Code 内で確認可能に

Copilot ステータスダッシュボードに、設定済みの追加 Copilot 予算のうち消費済みの割合が表示されるようになりました。設定した上限に達する前に使用量を把握し、利用を調整できます。詳細な使用状況の確認や追加利用量の管理は [Copilot 設定ページ](https://github.com/settings/copilot/features) から行えます。

---

### 言語モデルエディターからモデルプロバイダーをインストール可能に

BYOK（Bring Your Own Key）モデルに加え、拡張機能が独自のモデルプロバイダーを提供できます。これまでは Extensions ビューで `language-models` タグを検索する必要がありましたが、言語モデルエディターに **Install Model Providers** ボタンが追加され、モデルプロバイダーを提供する拡張機能を簡単に発見・インストールできるようになりました。インストール後、そのプロバイダーのモデルがモデルピッカーに表示されます。

詳細は [言語モデルのドキュメント](https://code.visualstudio.com/docs/agent-customization/language-models) を参照してください。

---

### 統合ブラウザのアドレスバーから Web 検索

**設定**: `workbench.browser.searchEngine`

統合ブラウザのアドレスバーにクエリを入力すると、設定済みの検索エンジンで検索が実行されるようになりました。VS Code を離れることなく情報を検索できます。使用する検索エンジンは `workbench.browser.searchEngine` 設定で変更可能です。

---

### 統合ブラウザのリモート接続経由でのブラウジング（プレビュー）

**設定**: `workbench.browser.enableRemoteProxy`

リモートワークスペースで統合ブラウザを開いた際、HTTP(S) トラフィックをリモート接続経由でプロキシできるようになりました。リモートマシンからのみアクセス可能なポートやサービスにセキュアに接続できます。プレビュー機能のため、`workbench.browser.enableRemoteProxy` 設定を有効にして試すことができます。

---

### エージェントによるフォワードポートとの連携改善

リモートワークスペースでポートフォワーディングを行っている場合、これまではポート番号の違いによりエージェントがブラウザを開く際に問題が発生することがありました。今回のリリースでは、エージェントがフォワード済みのポートをリクエストした場合（リモートプロキシが無効の場合）、URL が自動的に書き換えられ、エージェントに変更が通知されるようになりました。

---

### エンタープライズ向け：ネイティブ MDM による Copilot 管理設定の配信

管理者は、アカウントベースのエンタープライズ設定ファイルに加え、Windows および macOS のネイティブデバイス管理（MDM）チャネルを通じて GitHub Copilot の管理設定を配信できるようになりました。既存のデバイス管理ツールを使用して Copilot の設定を強制でき、ユーザーごとのサインインに依存せずにポリシーを適用できます。MDM 経由で配信された設定は VS Code 内でポリシー適用済みとして表示され、ローカルで上書きすることはできません。
