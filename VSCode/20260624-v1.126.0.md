# Visual Studio Code v1.126.0 (June 2026) — GitHub Copilot 関連アップデート

## リリース日

2026-06-24

## VS Code バージョン

1.126.0

## 概要

チャットセッション単位のコスト表示、モデルカスタマイズピッカーの統合、エージェントウィンドウでの複数チャット対応およびコードレビューフィードバック連携など、コスト管理とエージェント体験を強化するリリース。

---

## GitHub Copilot 関連の変更点

### セッション単位のコスト情報表示

チャットセッション全体のコストが確認できるようになりました。これまでは個々のターンごとのコストのみ表示されていましたが、セッション全体の消費クレジットが表示されることで、どのセッションが最もコストを消費しているかを把握しやすくなり、使用量の管理が容易になります。

---

### モデルカスタマイズピッカーの統合

言語モデルの設定を簡素化するため、コンテキストサイズと推論（思考）エフォートの制御が単一のモデルカスタマイズピッカーに統合されました。これまで2つの別々のドロップダウンで操作していた設定を、1つの場所からまとめて調整できるようになりました。

---

### モデルホバーの簡素化

モデルホバーが整理され、スキャンしやすくなりました。モデルの機能を示す簡潔な1ワードの説明が表示されるようになり、関連する設定に直接移動できるディープリンクボタンも追加されました。

---

### エージェントウィンドウでの複数チャット対応（プレビュー）

エージェントウィンドウのエージェントホストから開始した Copilot セッションで、複数のチャットを同時に管理できるようになりました。チャットは同じセッションとワーキングコンテキストを共有するため、1つのワークスペース内で複数の会話を並行して進めることができます。

例えば、メインのチャットが機能の実装中であっても、セッションツールバーの **New Chat**（`+`）ボタンから2つ目のチャットを開き、変更のレビュー、テストの作成、ドキュメントの執筆などを同時に行えます。各チャットは独自の会話履歴を保持し、タブを切り替えて中断した場所から再開できます。

チャットはウィンドウのリロード後も保持・復元されます。また、タブをダブルクリックするか、コンテキストメニューから **Rename** を選択することで、チャットのタイトルを変更できます。チャットのタイトルはセッションタイトルとは独立しているため、セッション名の変更がチャット名に影響することはありません。

---

### エージェントホストハーネスによるエージェント的コードフィードバック

エージェントウィンドウで生成されたコードに残したコメントが、エージェントホスト上に保存されるようになりました。これにより、エージェントは `listComments` や `resolveComments` などのサーバーサイドツールを使用してフィードバックを操作できます。コメントはローカルセッションではなくサーバー上に保存されるため、クライアントを切断した後もエージェントがフィードバックを処理できます。

エージェントは `addComment` ツールを使用してコメントを作成することも可能です。`/code-review` などのレビュースキルを実行すると、コードがレビューされインラインでコメントが追加されます。追加されたコメントを承認または削除してから、エージェントに送信して対応させることができます。

プルリクエストのレビューコメントも同様に機能します。PR レビューコメントを承認してエージェントに送信するか、エージェントにすべての PR コメントを解決するよう依頼できます。まだ承認していない PR コメントの解決をエージェントに依頼した場合、エージェントはまずコメントの閲覧許可をリクエストし、アクセスが許可された後にレビュー項目に対応します。
