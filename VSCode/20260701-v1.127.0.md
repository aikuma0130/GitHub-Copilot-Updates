# Visual Studio Code v1.127.0 (July 2026) — GitHub Copilot 関連アップデート

## リリース日

2026-07-01

## VS Code バージョン

1.127.0

## 概要

ブラウザツールのGA化、ターミナルコマンドのサンドボックス化、エージェントウィンドウのセッション整理機能強化、サブエージェントのコスト可視化など、エージェント体験の成熟と安全性を高めるリリース。

---

## GitHub Copilot 関連の変更点

### エージェントウィンドウ：グループによるセッション整理（プレビュー）

複数のエージェントセッションを同時に実行するとセッションリストが長くなりがちですが、カスタムグループを作成してセッションを整理できるようになりました。グループヘッダーを折りたたんでリストを整理し、重要なセッションに集中できます。各グループからは新しいセッションの開始や、すべてのセッションを完了済みにするクイックアクションが利用できます。

---

### エージェントウィンドウ：セッションリストのドラッグ＆ドロップ

セッションリストでドラッグ＆ドロップがサポートされ、以下の操作が可能になりました：

- セッションを上下にドラッグして並べ替え
- グループヘッダーやワークスペースヘッダーをドラッグしてリストを再配置
- セッションをグループにドロップして追加
- セッションを「ピン留め」セクションにドロップしてピン留め
- 複数セッションを選択してまとめて移動

---

### エージェントウィンドウ：チャット入力バナー

エージェントセッションにオープンなプルリクエストがある場合、チャット入力の上にバナーが表示され、CIの失敗やレビューコメントに即座に対応できるようになりました。

- **CI失敗時**: 失敗したチェック数が表示され、「Fix Checks」でエージェントに修正を依頼、「Reveal Checks」でChangesビューに表示
- **PRコメント受信時**: コメント数が表示され、「Address Comments」でエージェントに対応を依頼、「Reveal Comments」でエディタに表示

---

### エージェントウィンドウ：オンボーディングツアー（実験的）

エージェントの使い方を学ぶためのガイド付きウォークスルーがエージェントウィンドウに追加されました。主要な機能を紹介し、タスクの委任方法や生産性を高めるためのベストプラクティスを案内します。

---

### エージェントウィンドウ：エディタガターからのフィードバック

エージェントの変更をレビューする際、エディタのガター（行番号の左側）にホバーすると「Add Feedback」グリフが表示されるようになりました。クリックするとその行にコメントを残すことができ、エージェントに具体的なコード箇所を直接指示できます。

---

### エージェントウィンドウ：PRタイトルと説明の自動生成改善

エージェントウィンドウからプルリクエストを作成する際、セッションのコンテキストを使用してPRタイトルと説明が自動生成されるようになりました。これにより、セッションで行った作業をより正確に反映した説明的なPRが作成されます。

---

### マルチチャットセッションの改善

1つのエージェントホストセッション内で複数のチャットを管理するマルチチャットセッションに以下の改善が加えられました：

- **チャットの閉じる・再開・削除**: タブの「X」でチャットを非表示にし、「Conversations」ドロップダウンから再表示可能。コンテキストメニューから完全削除も可能
- **全チャットの進捗と変更の集約表示**: セッションヘッダーのChangesピルが全チャットの編集を反映し、いずれかのチャットが作業中であればセッション全体が進行中と表示
- **会話のフォーク**: マルチチャットセッション内で会話をフォークすると、新しいトップレベルセッションではなく同じセッション内のピアチャットとして作成

---

### /troubleshoot によるエージェント動作の診断

`/troubleshoot` コマンドがエージェントホストセッション（ローカルおよびリモート）の診断に対応しました。エージェントウィンドウのチャット入力で `/troubleshoot` と入力し、`#session` でセッションを選択して問題を記述することで、カスタム指示が無視される原因や応答が遅い理由などを調査できます。

---

### サブエージェントのクレジット表示

エージェントがサブエージェントに作業を委任した際のコストを確認できるようになりました。チャットレスポンス内のサブエージェントセクションにホバーすると、そのサブエージェントが使用したAIクレジットが表示されます。

---

### ターミナルコマンドのサンドボックス化（macOS / Linux）

macOSとLinuxで、エージェントが実行するターミナルコマンドのサンドボックス化が導入されました。コマンドはネットワークアクセスがブロックされ、ファイルシステムアクセスが制限された状態で実行されるため、エージェントが少ない承認プロンプトで作業できるようになります。サンドボックスの外で実行する必要がある場合のみ承認が求められます。Permissionsドロップダウンからこの機能を無効にすることも可能です。

---

### ブラウザツール（エージェント用）が正式リリース（GA）

エージェントが統合ブラウザでページを開き、コンテンツやコンソールエラーを読み取り、スクリーンショットを撮影し、選択・入力・ナビゲーションを行って作業を検証できるブラウザツールが正式にリリースされ、デフォルトで有効になりました。外部のMCPサーバーなしで利用可能です。

管理者はエンタープライズポリシーでブラウザツールを制御できます：`BrowserChatTools` ポリシーで完全に無効化するか、`ChatAgentNetworkFilter` と許可/拒否ドメインリストでエージェントがアクセスできるドメインを制限できます。

---

### 組み込み Ollama プロバイダーの非推奨化

Ollama が[公式 VS Code 拡張機能](https://marketplace.visualstudio.com/publishers/Ollama)を提供したことに伴い、組み込みの Ollama プロバイダーが非推奨になりました。ローカルの Ollama モデルをチャットで使用している場合は、公式拡張機能をインストールし、組み込みプロバイダーを削除してください。

---

### ファイルベースの管理対象 Copilot 設定配信（エンタープライズ）

管理者がディスク上のJSONファイルからGitHub Copilotの管理対象設定を配信できるようになりました。MDMに登録されていないマシンや、構成管理システムからファイルをプロビジョニングする方が容易な環境向けの機能です。

VS Code は以下のOS固有のパスから `managed-settings.json` を読み取ります（MDMまたはアカウントベースのエンタープライズ設定が存在しない場合のみ）：

- **macOS**: `/Library/Application Support/GitHubCopilot/managed-settings.json`
- **Linux**: `/etc/github-copilot/managed-settings.json`
- **Windows**: `%ProgramFiles%\GitHubCopilot\managed-settings.json`
