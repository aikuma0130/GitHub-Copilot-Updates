# Visual Studio Code v1.128.0 (July 2026) — GitHub Copilot 関連アップデート

## リリース日

2026-07-08

## VS Code バージョン

1.128.0

## 概要

マルチチャットエージェントセッションの強化、Copilot Vision の一般提供開始、OS レベルのキーボードショートカット対応など、エージェント機能とチャット体験を大幅に向上させるリリース。

---

## GitHub Copilot 関連の変更点

### マルチチャットセッションが Claude エージェントに対応

Agents ウィンドウの Claude エージェントホストセッションで、複数チャットがサポートされました。1つのセッション内に関連するチャットスレッドをまとめることで、アプローチの比較、既存ターンからの分岐、並行作業が可能になります。

各チャットは独自の履歴、タイトル、モデル選択を保持し、親セッションとともに再起動後も復元されます。ピアチャットはClaude セッション配下にグループ化され、個別のトップレベルセッションとしては表示されません。

---

### ワークスペース未選択での Agents ウィンドウチャット

Agents ウィンドウで、ワークスペースを選択せずにチャットを開始できるようになりました。プロジェクトに紐づかない質問に対して、**Chats** セクションにクイックチャットとして表示され、入力可能な状態で開きます。

ワークスペースなしのチャットでは、ワークスペース固有の **Changes** や **Files** サイドペインは表示されません。クイックチャットはリロード後も他のセッションとともに復元されます。

この機能はエージェントホストセッションでのみサポートされ、`chat.agentHost.enabled` 設定が有効である必要があります。

---

### 読み取り専用サブエージェントチャット（プレビュー）

エージェントがサブエージェントに作業を委任した際、そのトランスクリプトが読み取り専用のピアチャットとして表示されるようになりました。メインの会話を中断することなく、各ワーカーの進捗を追跡できます。

サブエージェントチャットは、Conversations メニュー、実行中サブエージェントチップ、または親トランスクリプト内のインラインサブエージェントピルから開けます。コンポーザーや変更アクションは省略され、ワーカートランスクリプトは表示専用のままとなります。

---

### Agents ウィンドウのチャット用キーボードショートカット

マルチチャットセッション内でのキーボード操作が強化されました。

- チャットの作成、最後に閉じたチャットの再オープン
- 次/前のチャットへの移動
- 開いているチャット間のクイック切り替え
- アクティブなチャットタブの閉じる/削除
- 検索可能なチャットピッカーの表示

これらのショートカットは Agents ウィンドウにスコープされ、チャット固有のアクションがない場合は既存のセッションレベルの動作にフォールバックします。

---

### Copilot Vision の一般提供開始

マルチモーダルサポートが VS Code で一般提供（GA）になりました。画像や PDF をチャットに貼り付け、ドラッグ＆ドロップ、またはコンテキストメニューから添付できます。エージェントがツールコールを通じて画像を読み取ることも可能です。

サポートされるフォーマットと利用可能な詳細については [GitHub changelog](https://github.blog/changelog/2026-07-01-copilot-vision-is-generally-available/) を参照してください。

---

### エージェントホスト Copilot セッションでの BYOK モデル（実験的）

**設定**: `chat.agentHost.byokModels.enabled`

Bring Your Own Key（BYOK）モデルをエージェントホスト上のセッションで使用できるようになりました。設定を有効にし、エージェントホストプロセスを再起動すると反映されます。

この機能は実験的であり、現在も活発に開発中です。

---

### カスタムエンドポイントモデルのサンプリングパラメータ設定

カスタムエンドポイントモデルごとに `temperature` と `top_p` を設定できるようになりました。厳密なパラメータ要件を持つプロバイダーとの連携に対応します。

モデルの JSON 設定に `modelOptions` オブジェクトを追加することで、デフォルト値のオーバーライドやパラメータの省略（`null` 指定）が可能です。Chat Completions、Responses、Messages 互換エンドポイントに適用されます。

---

### BYOK のデフォルトユーティリティモデル設定

**設定**: `chat.byokUtilityModelDefault`

BYOK モデルをメインエージェントモデルとして使用する際、チャットタイトル生成やコミットメッセージ生成などの組み込みユーティリティフローで使用されるデフォルト動作を変更できるようになりました。

メインエージェントモデルの使用、GitHub Copilot 提供モデルの使用、またはユーティリティモデルを使用しない、のいずれかを選択可能です。`chat.utilityModel` や `chat.utilitySmallModel` で設定されたモデルがこのデフォルトよりも優先されます。

---

### 特定チャットへのディープリンク

`vscode://` ディープリンクでセッション内の特定チャットに直接遷移できるようになりました。ワークスペースを先に開いてからセッションを手動で探す必要がなくなります。

Agents ウィンドウの **Open in VS Code** アクションも同じ動作を使用し、セッションのワークスペースフォルダとアクティブなチャットの両方を新しい VS Code ウィンドウで開きます。

---

### OpenTelemetry による Copilot テレメトリエクスポートの管理（Enterprise）

組織は GitHub Copilot が OpenTelemetry（OTel）データを送信する先を管理設定で指定できるようになりました。開発者が個別に `OTEL_*` 環境変数を設定する必要なく、承認済みのコレクターにテレメトリを送信できます。

管理設定の `telemetry` ブロックで以下を制御できます：

- OTLP エクスポートエンドポイントとプロトコル
- OTel サービス名とリソース属性
- エクスポーターヘッダー（認証トークンなど）
- プロンプトとレスポンスコンテンツのキャプチャ可否、および開発者による変更可否

管理値は常に優先され、環境変数やユーザー設定よりも上位となります。
