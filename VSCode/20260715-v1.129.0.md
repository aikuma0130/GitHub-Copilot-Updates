# Visual Studio Code v1.129.0 (July 2026) — GitHub Copilot 関連アップデート

## リリース日

2026-07-15

## VS Code バージョン

1.129.0

## 概要

専用エージェントホストプロセスの導入、Agents ウィンドウの新しいエディターパネル、`!` プレフィックスによるターミナルコマンド実行、エージェントホストでの BYOK モデルサポートなど、エージェントアーキテクチャを大幅に刷新するリリース。

---

## GitHub Copilot 関連の変更点

### エージェントホストの導入

VS Code でのエージェントセッションの動作が、専用の「エージェントホスト」プロセスを中心に再構築されました。エージェントホストは [Agent Host Protocol](https://microsoft.github.io/agent-host-protocol/)（AHP）に基づき、Copilot、Claude、Codex などのエージェントハーネスを実行する専用プロセスです。

セッションが独自のプロセスで動作するため、同じセッションを複数の VS Code ウィンドウから同時に接続・表示できます。エージェントホストの Copilot エージェントは [Copilot SDK](https://www.npmjs.com/package/@github/copilot-sdk) を使用しており、Copilot CLI、スタンドアロン GitHub Copilot アプリ、その他の Copilot 製品と動作・機能が統一されています。

オプトインするには `chat.agentHost.enabled` を有効にし、ハーネスドロップダウンからエージェントホストハーネスを選択します。

---

### Agents ウィンドウの新しいエディターパネル（実験的）

Agents ウィンドウに再設計された**エディターパネル**が導入されました。エディターと詳細エリアが共有タブバーを持つ1つのドッキングペインに統合され、エージェントの作業を確認する体験がメインエディターでの操作に近づきました。

新しいエディターパネルでは以下が可能です：

- ファイルや差分をドッキングされたエディター内でチャットの横に直接開く
- **Changes** ビューでインラインとサイドバイサイドの切り替え、全ファイルの展開/折りたたみ、コンパクトな差分表示が可能
- セッション切り替えやウィンドウリロード時にサイドペインの幅、開いているエディター、アクティブなエディター、ファイルごとの折りたたみ状態を復元

この機能は実験的なオプトインレイアウトです。有効にするには `sessions.layout.singlePaneDetailPanel` を設定してウィンドウをリロードしてください。

---

### セッション管理ツール（エージェントホストセッション向け）

エージェントホスト上で動作するエージェント（Copilot、Claude、Codex）が、セッション管理ツール群にアクセスできるようになりました。エージェントが他のセッションやチャットを列挙、作成、観察、操作できます。

これらのツールにより、エージェントは以下が可能です：

- セッションの一覧表示（ステータス、ワークスペース、変更内容を含む）
- 別のセッションの最近の会話の読み取り
- 新しいセッションまたは既存セッション内の新しいチャットの作成（サブタスクの委任用）
- 作成したセッションやチャットへのメッセージ送信

ツールがセッションを作成または対象とする場合、VS Code は **Open Session** ピルを表示します。別のセッションへのメッセージ送信は常に確認が求められ、エージェントは自身のチャットにメッセージを送信することはできません。

---

### Agents ウィンドウの改善

新しいセッション作成フローにいくつかの改善が加えられました：

- **新規セッションデフォルトの記憶**: 新規セッションピッカーが前回のエージェントモードと承認の選択を記憶し、次回作成時のデフォルトとして使用
- **ワークツリーチェックボックス**: フォルダとワークツリー分離をドロップダウンから選択する代わりに、単一の **New Worktree** チェックボックスに変更。Git ワークツリー分離でセッションを実行するかどうかをチェックボックスで制御

---

### `!` プレフィックスによるターミナルコマンド実行

チャットメッセージの先頭に `!` を付けることで、その内容をターミナルコマンドとして実行できるようになりました。この機能はエージェントホストセッションのエディターウィンドウと Agents ウィンドウの両方で動作します。

---

### エージェントホスト Copilot ハーネスでの BYOK モデル

Agents ウィンドウでエージェントホスト上の **Copilot** ハーネスを選択している場合に、[Bring Your Own Key（BYOK）モデル](https://code.visualstudio.com/docs/agent-customization/language-models#_bring-your-own-language-model-key)を使用できるようになりました。

---

### プロンプトファイルからスキルへの移行（実験的）

プロンプトファイル（`*.prompt.md`）はカスタムスラッシュコマンドを記述するために使用されますが、ローカルエージェントハーネスでのみサポートされています。他のハーネスではスキルとしてスラッシュコマンドを表現するため、互換性のためにすべてのプロンプトファイルをスキルに移行することが推奨されます。

`chat.customizations.promptMigration.enabled` を有効にし、エージェントホスト上のハーネスを選択している場合、移行可能なプロンプトファイルがあれば AI Customizations の概要に「Migrate Prompts」エントリが表示されます。

移行インターフェースでは以下が可能です：

- ワークスペース（`.github/prompts/`）およびユーザーデータの両方からプロンプトファイルを表示
- 選択したファイルをスキルに移行し、新しく作成されたスキルを開く

---

### GitHub Enterprise での Copilot 認証サポート（エージェントホスト）

GitHub Enterprise（GHE）インスタンスを通じて GitHub Copilot アクセスが提供されている開発者が、VS Code でサインインして Copilot を使用できるようになりました。

以前はエージェントホストの Copilot 認証が github.com のみをサポートしていたため、GHE ベースの Copilot サブスクリプションではサインインを完了できませんでした。今回のリリースで、VS Code が GitHub Enterprise インスタンスに対して Copilot を認証できるようになり、OAuth フローと Copilot アクセストークンのリクエストが選択した GHE ホストに対して実行されます。

この機能はエディターウィンドウと Agents ウィンドウの両方で、Copilot と Claude の両エージェントで動作します。`chat.agentHost.enabled` が有効である必要があります。
