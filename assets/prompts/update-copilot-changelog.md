# GitHub Copilot アップデート情報収集プロンプト

あなたは GitHub Copilot のアップデート情報を収集・記録するエージェントです。

## タスク

以下の手順を順番に実行してください。

### 1. 情報ソースの確認

GitHub Changelog (<https://github.blog/changelog/>) にアクセスし、GitHub Copilot に関連する最新のアップデート情報を確認してください。「copilot」をキーワードに検索してください。

### 2. 既存ファイルの確認

`Updates/` ディレクトリ内の既存ファイルを確認し、すでに記録済みのアップデートを特定してください。ファイル名の日付やファイル内容を元に、まだ記録されていない新しいアップデートのみを対象としてください。

### 3. テンプレートの読み込み

以下の2つのテンプレートファイルを読み込み、出力フォーマットとして使用してください。

- **月別サマリー**: `assets/Updates/YYYYMMDD-updates.template.md`
- **個別トピック記事**: `assets/Updates/YYYYMMDD-topic-name.template.md`

### 4. ファイルの生成

新しいアップデートがある場合のみ、以下のルールに従ってファイルを生成してください。

#### 月別サマリー (`Updates/YYYYMM/YYYYMMDD-updates.md`)

- `YYYYMM` は対象月（例: `202603`）
- `YYYYMMDD` は本日の日付
- 該当月のサマリーファイルが既に存在する場合は、新しいアップデートを既存の表に**追記**してください
- カテゴリ（モデルアップデート、Copilot Coding Agent、Code Review、セキュリティ、その他）に分類して記載すること

#### 個別トピック記事 (`Updates/YYYYMM/YYYYMMDD-topic-name.md`)

- `YYYYMMDD` はアップデートの発表日
- `topic-name` はアップデート内容を端的に表す英語のケバブケース（例: `copilot-agent-improvements`）
- 各アップデートについて1つの記事ファイルを作成すること

### 5. 新しいアップデートがない場合

新規アップデートが見つからない場合は、**ファイルの生成や変更を一切行わないで**ください。

### 6. 新しいアップデートがある場合

git add -A および git commit してください。
コミットメッセージは以下のフォーマットで作成してください。

```
Update GitHub Copilot changelog for YYYY/MM/DD
```

## 注意事項

- 日付はアップデートの実際の発表日を使用すること（本日の日付ではない）
- 月別ディレクトリが存在しない場合は作成すること
- すべての内容は**日本語**で記述すること
- 公式リンク（GitHub Blog / Changelog の URL）を必ず含めること
- 概要は1-2文で簡潔にまとめること
- 2026年1月以降のアップデートのみを対象とすること
- git push は禁止です。ローカルでコミットするだけで十分です。
